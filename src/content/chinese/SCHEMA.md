# 중국어 레퍼런스 콘텐츠 형식 (Chinese Reference Schema)

중추 레퍼런스 = **문법**(`grammar/<level>.js`) + **어휘**(`vocab/<level>.js`) + **문형 사전**(`bunkei/<level>.js`).
레벨: OT(입문 상식) → H1 → H2 → H3 → H4 → H5 → H6 (HSK 1급~6급에 대응).
`index.js`가 전체를 레지스트리로 묶음. 콘텐츠는 코드가 단일 소스 (DB 불필요).

> **레벨 표기**: 내부 key는 `OT`/`H1`~`H6` (파일명·URL용 ASCII), 화면 표기는 `short`로 **OT / 1급 ~ 6급**.

대상 독자: **한국어 모어 화자**, 영어 B1 수준 가정.
- 설명은 전부 한국어. 부드러운 해요체 (강의 lessons 톤과 동일).
- **중국어의 최대 무기 = 공유 한자어**. `hanja` 콜아웃으로 "한국 한자음 ↔ 중국어 단어"를 적극 연결 (예: 学生 xuéshēng = 학생, 时间 shíjiān = 시간). 단, 의미가 어긋나는 가짜동족어(老婆 lǎopo = 아내, '노파' 아님)는 `pitfall`로 경고.
- 한국인이 특히 헷갈리는 지점(SVO 어순, 성조, 양사, 把구문 등)은 `pitfall`로 명시적으로 처리.

## 핵심 필드 규약 (중국어)
- 원문 = **`zh`** (간체자), 발음 = **`pinyin`** (성조 부호 포함: nǐ hǎo).
- 어휘·문형·예문 모두 `zh` + `pinyin` 쌍. (다른 언어의 `fr`+`ipa`, `ja`+`yomi`에 대응)
- 병음은 표준 성조 부호로(mā má mǎ mà ma). 경성은 부호 없이(예: 妈妈 māma, 朋友 péngyou).

## 문법 챕터 (grammar/<level>.js)

`export default [ chapter, chapter, ... ]` — order 순 배열.

```js
{
  slug: 'h1-04-negation',       // 필수. 전체에서 유일. '<level소문자>-<번호>-<영문키워드>'
  level: 'H1',                  // 'OT'|'H1'|'H2'|'H3'|'H4'|'H5'|'H6'
  order: 4,

  // ── 제목 체계: 훅이 주인공, 용어는 라벨 ──
  title: '"안 먹어요"를 두 글자로',          // 훅 (8~20자) — 배우면 할 수 있는 말/깨달음
  topic: '부정 不・没',                      // 문법 용어 라벨 — 목록 우측·헤더 보조줄·SEO
  titleFr: '否定',                          // 원어(한자) 제목 — 필드명은 titleFr이나 중국어 한자 표기를 넣음

  summary: '한 줄 요약 — 챕터 목록 카드에 표시',
  duration: '약 8분',

  sections: [
    {
      heading: '소제목',
      pattern: '不 + 동사/형용사',            // 핵심 패턴 — 원어+한국어 라벨 혼용, + / → 활용
      patternKo: '현재·습관·미래의 부정',       // 선택: 한 줄 글로스

      body: '한국어 설명. 빈 줄(\\n\\n)로 문단 구분. **굵게** 마크업만 지원.',

      // 예문 — 해당 레벨까지 배운 어휘·문법만 사용
      examples: [
        { zh: '我不喝咖啡。', pinyin: 'wǒ bù hē kāfēi', ko: '저는 커피를 안 마셔요.', note: '선택: 짧은 부가 설명' },
      ],

      // 표 (선택)
      table: {
        caption: '인칭대명사',
        headers: ['단수', '복수'],
        rows: [ ['我 wǒ 나', '我们 wǒmen 우리'], ['你 nǐ 너', '你们 nǐmen 너희'] ],
      },

      // 콜아웃 (각각 선택, 문자열, **굵게** 지원)
      pitfall: '한국인이 자주 틀리는/헷갈리는 점',     // 🚨 주황 띠
      vsKo:    '한국어와의 비교',                    // 🇰🇷 파랑 띠
      hanja:   '한자어 연결 — 한국 한자음 ↔ 중국어',  // 🈶 띠 (중국어의 핵심 무기)
      tip:     '보너스 팁·암기 요령',                // 💡 보라 띠
    },
  ],
}
```

### 가이드라인
- 챕터당 sections 3~5개. 섹션당 examples 2~5개.
- **패턴 우선 원칙**: 구체적 문형을 가르치는 섹션엔 반드시 `pattern` 한 줄.
- **body 간결성**: 문단 최대 2개, 문단당 2~3문장. 핵심 용어·형태는 **굵게**.
- **예문은 해당 레벨에 맞는 어휘·문법만** — H1 챕터에 把구문 금지.
- **예문 pinyin 필수** (OT~H3 100%, H4+ 권장).
- 콜아웃은 챕터 전체에서 3~5개. `hanja`를 적극 활용하되 한 섹션에 몰지 말 것.
- `pitfall` 우선순위 1: **어순**(중국어는 SVO, 한국어는 SOV), **양사**(个/本/张...), **성조**, **是 vs 형용사술어**(형용사 앞에 是 안 씀), **把/被**.

### JS 문자열 규칙
- 콘텐츠 문자열은 큰따옴표 기본. 한자·병음에 작은따옴표 충돌 없음.
- 검증: `node -e "import('./src/content/chinese/grammar/h1.js').then(m=>console.log(m.default.length))"`

## 어휘 (vocab/<level>.js)

```js
export default {
  level: 'H1',
  title: '1급 기초 어휘',
  desc: '한 줄 소개',
  themes: [
    {
      name: '인사와 사람',
      words: [
        {
          zh: '学生',
          pinyin: 'xuésheng',
          ko: '학생',
          pos: '명사',                 // 명사 | 동사 | 형용사 | 부사 | 양사 | 대사 | 개사 | 조사 | 표현 등
          hanja: '학생(學生) — 한자가 그대로',   // 선택: 한국 한자음 연결 (도움될 때만, 적극 활용)
          ex: { zh: '我是学生。', pinyin: 'wǒ shì xuésheng', ko: '저는 학생이에요.' },  // 선택
        },
      ],
    },
  ],
}
```

### 가이드라인
- 테마 4~8개. 테마당 단어 8~20개. (icon 필드는 쓰지 않음 — 이모지 정리됨)
- `hanja`는 한국 한자음으로 뜻을 유추할 수 있을 때 적극 (가짜동족어는 경고로).
- 레벨 어휘량 가이드: H1 ~100 / H2 ~100 / H3 ~90 / H4 ~80 / H5 ~70 / H6 ~60.
- **챕터 예문에 등장하는 내용어는 같은 레벨 이하 어휘 사전에 수록** (전 언어 공통 규칙).

## 문형 사전 (bunkei/<level>.js) — HSK 전수 커버 레이어

챕터가 '이해'라면 문형 사전은 '전수 검색'. 해당 레벨의 핵심 구문·표현을 빠짐없이 수록.

```js
export default {
  level: 'H1',
  title: '1급 문형 사전',
  desc: '한 줄 소개',
  themes: [
    {
      name: '부정·금지',
      items: [
        {
          pattern: '不 + V',                       // 문형 (대표 표기)
          conn: '不 + 동사/형용사',                  // 구조·접속
          ko: '~하지 않다 (현재·습관·의지의 부정)',     // 한국어 대응 (간결)
          ex:  { zh: '我不喝酒。', pinyin: 'wǒ bù hē jiǔ', ko: '저는 술을 안 마셔요.' },
          ex2: { zh: '他不是老师。', pinyin: 'tā bú shì lǎoshī', ko: '그는 선생님이 아니에요.' },  // 필수 — 항목당 예문 2개
          note: '没는 과거·완료의 부정 — 不과 구별',     // 선택 — 한 줄 주의점
          ch: 'h1-04-negation',                    // 선택 — 이 문형을 다루는 챕터 slug
        },
      ],
    },
  ],
}
```

- 모든 항목 필수: pattern · conn · ko · ex · ex2 (각 zh+pinyin+ko). note/ch는 선택.
- **pattern에 한국어 괄호 금지** — 뜻·용법 구별은 ko나 note. 동음 문형은 하나로 합쳐 ko에 ①② 병기.
- **pattern 구분자 규칙** — 병렬 형태는 `・`로(렌더 시 줄바꿈): `不・没`. `/`는 한 형태 안의 낱말 교체에만(`这/那`).
- **같은 문형의 긍정/부정 분리 금지** — 하나로 묶고 ex=긍정, ex2=부정.
- 예문 pinyin 필수. 예문은 그 레벨까지의 어휘·문법으로 짧게.
- 테마는 기능별 그룹 8~16개. 목표 수: H1 80+ / H2 90+ / H3 100+ / H4 100+ / H5 80+ / H6 60+.
