# RFC — 한국어 문체 계약 v1 (설명·번역 레지스터 통일)

상태: 채택(2026-07-24). 재평가 R2에서 드러난 트랙 간 문체 루브릭 불일치(프랑스어는
번역 제외·중국어는 예문 번역 3,845건을 결격 사유로 계상)를 계약으로 통일한다.

## 1. 계약

| 영역 | 문체 | 비고 |
|---|---|---|
| 산문 설명(body·tip·pitfall·vsKo·etym·summary·explain·note 등) | **해요체** | 기존 계약 재확인 — 4트랙 소탕 완료(#593·#609) |
| 예문·대화의 ko 번역 — 평서 서술 | **해요체** | 트랙 정합: fr·ja·en은 이미 해요체 관례, **zh만 -다체 3,845건 → 이행 대상** |
| 예문·대화의 ko 번역 — 원문 레지스터가 정보인 경우 | **원문 따라감** | 반말 원문→반말 번역(Tu 대화·타메구치), 격식 원문→합쇼체(비즈니스·안내방송), 문어·격언·신문 헤드라인→-다체 유지 |
| 인용 글로스('~라고 합니다' 등 뜻풀이 인용) | 원형 유지 | 문형이 가르치는 표현 자체 |
| 퀴즈 지시문 | 해요체 | 기존 관례 |

판정 원칙: **번역의 문체는 콘텐츠다** — 원문이 레지스터를 가르치는 예문(존댓말
장면·반말 장면)은 그 레지스터를 보존하고, 중립 평서 예문만 해요체로 통일한다.

## 2. zh 예문 번역 이행(-다체 3,845건)

- 방식: 규칙 변환기(종결어미 표: ~한다→해요·~했다→했어요·~이다→예요/이에요·
  ~된다→돼요·~ㄴ다/는다→활용형) + **변환 불가·불확실 잔여는 수동 목록화** →
  전수 diff 토큰 검수(#609 확립 방법) → 배치 PR 분할(팩 단위, 회당 ≤800건).
- 제외: 격언·문어 예문(성어 뜻풀이 등 -다체가 자연스러운 경우 — 변환기에서 태그로 배제).
- 게이트: 각 배치마다 받침+예요·어간 비문·불규칙 활용(와요·써요·들어요 등) 스캔 0.

## 3. 루브릭 후속

재평가 R3부터 이 계약을 기준으로 계상한다(트랙 간 동일 기준).
