# 기획 — 독해 트랙 파일럿 "도쿄 도착" (JA N5) · 확정판

> **한 줄 요약.** 코스의 축을 "챕터 완주"에서 "글 이해 완주"로 바꾼다. 실존 지명 기반 여행
> 이야기를 쉬운 것부터 차례로 읽고, **글 전부 + 최소 드릴 통과 = 급 문형 전수 커버**가
> 코드로 증명되도록 설계한다. JA N5 한 급으로 끝까지 만들어 보고 확산을 결정한다.
>
> 초안을 레드팀 에이전트로 스트레스 테스트(반박 14건, H6·M6·L2)해 전면 반영한 판이다.
> 반영 지점은 본문에 (RT-n)으로 표기.

## 0. 문제 정의 (오너 발견)

챕터는 급당 5~14개로 적은데 실제 학습량(어휘 700~2,000 + 문형 60~250 + SRS 정착)은 몇 달치다.
챕터가 UI에서 "코스"로 보이므로 하루 1챕터씩 열흘이면 급을 끝낸 착각이 생기고, 통과 판정도
직후 80% 퀴즈라 단기기억으로 뚫린다. 분량은 표준(JLPT/CEFR/HSK) 캘리브레이션이 돼 있으나,
**진도의 단위가 시간·정착과 무관한 것이 병인.**

## 1. 설계 원칙 (레드팀 반영 후)

- **정직한 완주 등식 (RT-6·7)**: "글을 다 읽으면 문형을 다 이해"가 아니라
  **"글 전부 + 최소 드릴 = 문형 전수 커버"**. N5 125문형 중 4지선다 독해로 판별 불가한
  군(조사 발음규칙 は/を/へ, 문말 뉘앙스 ね/よ/でしょう, 최소쌍 は·が / に·で, 조수사)은
  드릴이 정규 경로다. **드릴 비율 상한 15%(≈19문형)** — 초과하면 재설계.
- **i+1의 결정성 범위 한정 (RT-4)**: 결정적 게이트는 "신규 문형이 본문에 실재"까지만.
  "미도입 문형 부재"는 조사류(1글자 표면형이 편재)에서 원리적으로 문자열 검증이 불가하므로
  **P4 전수 감사 항목으로 강등**하고, 조사 테마는 초반 2~3글에서 일괄 도입한다.
- **서술 형태 트레이드오프의 명시 확정 (RT-5)**: 여행기는 본질적으로 과거·て연결 서술을
  요구하는데 모듈 순서상 둘 다 후반이다. 해법: 초반 글은 **현장 대화·안내판·실시간 진행**
  프레임(현재형이 자연스러운 장르)으로 쓰고, 부득이한 て형 정형구(〜てください 등)는
  "통문장 표현"으로 예외 도입(문형 미계상, 사전 카드 링크만). 과거 서술은 〜ました 도입
  이후부터 허용. 이 규칙은 P0 시퀀스 표에 글별로 명시한다.
- **상황 필연**: 문형은 여행 상황이 요구해서 등장한다. 문형을 위해 상황을 비틀지 않는다.
- **어휘 전수 포함은 비목표**(오너 확정): 글은 어휘를 노출하고 정착은 FSRS가 맡는다.
- **사실성 프레임**: 실존 장소 + 허구 주인공. 랜드마크급 사실만 쓰고 세부 정보(영업시간·
  가격·소요시간)는 금지.
- 기존 헌법 유지: XP 금지, 요미는 LLM이 만들지 않는다, 모든 생성물은 검증 게이트 통과 후 입고.

## 2. 파일럿 범위 — "도쿄 도착" (JA N5)

| 항목 | 내용 |
|---|---|
| 문형 | bunkei N5 전량 **125개** — P0에서 "독해 판별 가능 / 드릴 필요" 이분류(RT-6), 드릴 ≤15% |
| 글 | **~30편** — 초반 4~6문장(신규 문형은 적게), 후반 8~12문장. 글별 부하 표로 관리(RT-13) |
| 동선 | 하네다 → 전철 → 호텔 → 편의점 → 식당 → 시부야 → 아사쿠사 → 우에노 → … → 도쿄역 |
| 어휘 제약 | N5 어휘 703 + **"여행 코어" 화이트리스트(상한 40)** + 지명 고유명사(RT-1) |
| 학습 시간 | 글당 5~8분 → 하루 1글 기준 **약 5주** — 시간 단위가 명시된 최초의 커리큘럼 |
| 챕터 | 삭제하지 않는다 — 코스 축에서 내려와 참조로 남는다 |
| 월드 매핑 | v1.5(파일럿 통과 후) — 데이터에 `place` 필드만 확보 |

### R1 하드 게이트: 어휘-동선 매핑 표 (RT-1 — 최우선 리스크)

레드팀 실측: 동선 필수 상황어의 다수가 703 풀 밖이다(空港·入国·パスポート·ホテル·予約·
コンビニ·メニュー·注文·会計·改札 등 17+). 따라서 **글을 한 편이라도 쓰기 전에**,
30글 각 장면의 필수 명사를 703에 매핑하는 표를 만들어:
- 미커버어는 "여행 코어" 화이트리스트로 승격하되 **총 40개 하드캡** (표제어·요미·뜻을
  갖춘 정식 어휘 카드로 함께 입고 — 몰래 지나가는 단어 금지),
- 캡을 넘기면 해당 장면을 풀 내 어휘로 표현 가능한 장면으로 교체.
**이 표가 캡 안에서 안 닫히면 착수 금지(하드 킬).**

## 3. 콘텐츠 스키마

`src/content/japanese/reading/n5_tokyo.js`:

```js
export default {
  track: 'n5-tokyo', title: '도쿄 도착', level: 'N5', estWeeks: 5,
  texts: [{
    id: 'n5-tokyo-01', order: 1,
    place: { name: '하네다 공항', ja: '羽田空港', landmark: 'airport' },
    title: '...', situation: '입국 심사',
    frame: 'dialogue',                        // dialogue|signage|narration(past) — RT-5 서술 규칙
    newPatterns: ['〜です', '〜ですか'],        // bunkei pattern 정확 일치, 드릴군 제외
    setPhrases: ['〜てください'],              // 예외 도입 통문장 표현(문형 미계상) — RT-5
    body: [{ ja, yomi, ko }, ...],
    questions: [
      // pattern 문항 — distractor는 "같은 문장의 다른 조사/활용형"(최소쌍 고정, RT-6·14):
      // 어휘 매칭으로 못 풀고 문형 지식이 있어야 풀린다.
      { type: 'pattern', pattern: '〜です', q, choices, answer, why },
      // content 문항 — 통과 게이트에서 제외, 이탈·품질 지표 전용(RT-14)
      { type: 'content', q, choices, answer },
    ],
  }, ...],
  drills: [ // 드릴군 문형(조수사·발음규칙·뉘앙스·최소쌍) — 정규 통과 경로(RT-6·7)
    { patterns: ['〜本', '〜枚', ...], style: 'form-choice', items: [...] },
  ],
}
```

- 검증 링크: 전 글 `newPatterns` ∪ 전 드릴 `patterns` = **125 전수, 중복 도입 0, 끊어진 ref 0**.
- **yomi**: kuromoji 파생 + check-furigana(정렬 검사) + **지명·다음자 요미 화이트리스트 대조
  검사 신설**(RT-9 — check-furigana는 정렬만 보고 오독을 못 잡는다. 羽田·浅草 등 장소 요미는
  정답 사전을 하드코딩해 kuromoji 출력과 대조).

## 4. 문형 사전 풍부화 (설명 이관)

- bunkei 아이템에 옵션 필드 `explain`(2~4문장 — 챕터의 한국어 화자 설명 이식),
  `contrast`(한국어 대조 1줄) 추가. 기존 `note`·`ch` 유지.
- 요약이 아니라 **이식**: 어순·조사 비교, 니모닉, ×行きますです류 함정 경고를 그대로 옮긴다.
- 글 뷰어에서 신규 문형 탭 → 사전 카드(explain) — "설명이 필요한 순간에 설명이 온다".
- **공수 정직화(RT-11)**: 125건 이관은 Opus 수작업 규모 — R1이 아니라 **독립 웨이브(R1.5)**.

## 5. 학습 흐름·데이터 (레드팀 정정 반영)

- **트랙 UI**: 글 목록(동선 순·진행 마커·순차 잠금) → 글 뷰어(문장별 원문→요미→뜻 계층,
  단어 탭→FSRS) → 문항. **뷰어·문항은 신규 컴포넌트**다(RT-11 — RefPatternCheck는 80%
  통과·3단계 구조라 재사용 불가, 셔플·채점 유틸만 재사용).
- **통과 판정**: pattern 문항 전부 정답(content는 지표 전용). 오답 시 해당 문장 재독 후
  재시도. 문항 distractor가 최소쌍 고정이라 어휘 매칭·찍기 내성이 있고, 시도 횟수 기록.
- **기록**:
  - 글 통과 = `user_ref_progress` upsert — 단, **독해 전용 readKey(`ja_reading_texts`)와
    slug 접두사(`rt:`)로 챕터 진도와 분리**(RT-12 — 같은 Set에 섞이면 챕터 완주율 오염).
  - 문항 채점 = `review_events`(source:`'reading'`, item_key: 문형 pattern,
    detail.{text_id, qtype:'pattern'|'content'}).
  - **rung은 미해당**(RT-3 — rung은 vocab 전용 사다리. reading 이벤트는 `computeWeakness`
    (약점 집계)·`computeEwma`(난이도 신호)만 소비하며 그 연결도 R3 작업으로 명시).
- **재검증 = 글 단위**(RT-2 — bunkei `ch`는 36/125가 부재, 13/125가 N4·N3 링크라 문형→챕터
  재검증은 성립 불가): 글 통과 시 `grammar_review`에 **글 slug**를 등록하고, /review/grammar에
  reading 분기(글 문항 재출제)를 추가한다(R3 스코프에 명시). D7 지표도 "글 재독 통과율"로 정의.
- **진도 표기**: "글 12/30 · 문형 58/125 · 예상 잔여 ~2.5주".

## 6. 생성 파이프라인 (모델 배분)

```
P0 시퀀스·분류 설계(Opus 초안 → Fable 게이트) → P1 결정적 준비(코드)
→ P2 글·드릴 생성(Sonnet 함대) → P3 결정적 검증(코드)
→ P4 전수 감사(30글 전수, 정량 루브릭) → P5 조립·PR(Fable)
```

### P0 산출물 (전부 통과해야 P2 착수 — RT-1·6·7·13)
1. **어휘-동선 매핑 표** (§2 하드 게이트)
2. **문형 이분류 표**: 독해 판별 가능 / 드릴 필요 — 드릴 ≤15% 확인
3. **시퀀스 표**: 글별 (신규 문형 수, 누적 문형 수, 문장 수, frame, setPhrases) — 초반 글이
   신규 상한을 실제로 지키는지 수치로 게이트(RT-13)

### 모델 배분 (공수 재산정 — RT-8·11)

| 모델 | 역할 | 규모 |
|---|---|---|
| **Fable 5** | 기획·P0 게이트·P3 지휘·최종 검수. 생성 노동 금지 | 오케스트레이션만 |
| **Opus 4.8** | ① P0 3표 초안 ② explain 이관 125건(R1.5) ③ P4 전수 감사 ④ 트랙 UI(뷰어·문항·잠금) ⑤ ja 검증기 설계 | 5~6기 |
| **Sonnet 5** | P2 글 생성(1기 4~5글) + 드릴 생성 + UI 보조 | 8~10기 |
| **코드** | 아래 P3 전부 | — |

### P3 결정적 검증 (실효 범위 정직화 — RT-4·8·9)
1. `newPatterns`∪드릴 = 125 전수·중복 0·끊어진 ref 0
2. 신규 문형 본문 실재(어간 매칭 — cloze 검증 방식 재사용)
3. **ja 내용어 커버리지 검사기 신설**: kuromoji 세그멘테이션+원형화 기반 — 기존 en/fr
   검사기는 공백 언어 전용이라 "확장"이 아니라 **신규 구축**임을 명시(RT-8). 요미 파생
   스택(derive_yomi)의 kuromoji 배선 재사용.
4. kuromoji 요미 파생 → check-furigana + **지명·다음자 요미 화이트리스트 대조**(RT-9)
5. 문항: 스키마·정답 실재·distractor≠정답·pattern 문항의 신규 문형 전수 커버·
   **distractor 최소쌍 규칙**(같은 문장의 조사/활용 치환) 형식 검사
6. ~~미도입 문형 부재~~ → P4 감사 항목(조사류 원리적 한계 — RT-4)

### P2 규칙(청크 프롬프트 고정)
사용 가능 문형 = 누적 목록만 / 신규 문형 각 2회 이상 자연 등장(드릴군 제외) / 어휘 =
703+여행 코어 40+지명만 / frame 준수(초반 dialogue·signage) / **yomi 빈칸** / 세부 사실 금지 /
문항 동시 생성(pattern 문항은 최소쌍 distractor 규칙 명시).

## 7. 측정과 킬 기준 (판정 가능하게 재설계 — RT-10)

**측정**: 글당 이탈률·평균 재시도(불량 글 탐지) / **글 단위** D7 재독 통과율 /
트랙 진입→완주 퍼널. 전부 review_events + user_ref_progress 유도.

**킬 기준**:
1. **R1 게이트**: 어휘-동선 표가 화이트리스트 40캡 안에서 안 닫히면 착수 금지.
2. **P4 정량 감사**: 문장별 자연성 루브릭 1~4점, "3 미만 문장이 있는 글" 비율 20% 초과 →
   해당 글 재생성 1회 → 재실패 시 중단. 판정은 **생성(Sonnet)과 다른 모델(Opus) + 문화
   정확성 별도 체크리스트**(관대 편향 완화).
3. **오너 실학습**: 앞 3글이 아니라 **위험 구간 포함 대표 5글**(조수사 드릴 1 + 과거 서술
   글 1 포함)로 판정 — 실패 사유가 킬 창 안에 들어오게(RT-10).
4. 통과 시: FR A0~A1 2차 파일럿(비한자 언어 일반성) → 4언어 확산 + 월드 매핑(v1.5).
