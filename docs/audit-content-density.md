# 26도시 콘텐츠 밀도 감사

## 결론

현재 26도시는 POI 349개와 설명문 100% 커버리지를 갖췄지만, 플레이 가능한 장치의 분포는 크게
기울어 있다. 특히 **보르도·스트라스부르**는 학습 도어·TRANSIT·transitPoints·NPC가 모두 0이고,
**서울·코트다쥐르·부산·레만호 연안**은 전체 맵 면적 대비 공간 앵커가 가장 성기다. 또 도시 진입
스탬프는 레거시 `WORLD_NODES`에 속한 7도시만 대상이며, 지역 오버월드 19도시는 현재 앨범 대상이
아니다. 이 문서는 실측과 제안만 담으며 runtime·registry·verifier·DB 구현은 하지 않는다.

## 측정 기준과 재현성

- 정본: `origin/main` `df6ce0e70df65e108fae92c91e6b67a6e81cb7ec`, Node `v22.23.1`.
- 도시 집합·순서: `CITY_MAPS` 26개. POI는 각 `cities/<id>.geo.js`의 `GEO.pois` 길이로 센다.
- 도어: runtime `city.nodes` 중 `chapter` 문자열이 있는 학습 진입점. 레거시 일본어 NPC형 도어도
  실제 같은 라우팅을 열기 때문에 포함한다. 따라서 도어와 NPC 열은 서로 배타적이지 않다.
- TRANSIT·transitPoints·props·NPC: 각각 `city.transit`, `city.transitPoints`, `city.props`,
  `city.nodes[kind=npc]`의 길이.
- desc 평균: desc가 있는 runtime node 전체의 Unicode code point 수 산술평균. 26도시 모두
  `descCoverage = nodes/nodes`라 누락 설명은 없다.
- 스탬프: `StampAlbum`이 소비하는 `WORLD_NODES` 중 `gate.type=city`, `gate.to=<city id>`,
  `noStamp !== true`인 도시 진입 노드가 있으면 `예`. 지역 오버월드의 `REGIONAL_WORLD_NODES`는
  현재 앨범 집합 밖이며 `noStamp: true`다.
- 면적: `cols × rows × metersPerTile²`. 공간 앵커는 `nodes + transitPoints + props`의 **서로 다른
  tile** 수이며, 밀도는 `앵커/면적(km²)`. 수면도 포함한 전체 bbox 면적이므로 장거리 벨트형 지도는
  낮게 나오며, 이는 이동 중 비어 보이는 정도를 보는 지표이지 콘텐츠 품질 점수는 아니다.
- 동일 one-shot Node 측정기를 두 번 실행한 JSON은 byte-identical. SHA-256
  `eb03d6e0f6873700778186448826606f0dfefca6205c1e1dc2e8ad6b4faeff07`.
- 전수 측정기: 0.81초, max RSS 743,948,288 bytes, peak footprint 641,708,296 bytes, swap 0.

단순히 POI로 채우기보다, 서울·부산·코트다쥐르의 빈 이동 구간과 캔버라의 낮은 장치 다양성을 먼저
보는 것이 타당하다. 몽생미셸은 POI 4뿐이지만 4m 정밀 티어와 7.28km²의 압축 지도라 1.373으로 가장
조밀하며, TRANSIT 0은 섬·조수·수도원 장면 구조상 결함으로 보지 않는다.

### 장치 공백

- 도어 0(11): `busan`, `seoul`, `cote-dazur`, `brussels`, `beijing`, `brisbane`,
  `canberra`, `melbourne`, `lyon`, `bordeaux`, `strasbourg`.
- TRANSIT 0(3): `mont-saint-michel`, `bordeaux`, `strasbourg`.
- transitPoints 0: 17/26. 특히 도선·버스 등 비철도 이동 장치가 있는 9도시에 집중돼 있다.
- NPC 0: 22/26. NPC 9개는 후쿠오카 3·도쿄 3·오사카 2·교토 1에만 있다.
- props 0: 몽생미셸 1곳뿐이다. 이곳은 전용 수도원/조수 장면이 별도 시각 장치를 제공한다.
- 도시 진입 스탬프 0: 지역 오버월드 19도시. 현재 대상은 후쿠오카·도쿄·오사카·교토·부산·
  서울·가와구치코 7곳이다.

## 균형 회복 제안 우선순위

1. **P0 — 보르도·스트라스부르의 플레이 장치 공백 해소**
   - 두 도시 모두 도어·TRANSIT·transitPoints·NPC가 0이다. 이미 존재하는 POI/역을 단일 진실원으로
     삼아 불어권 학습 도어 세트와 실제 도시축에 맞는 TRANSIT을 먼저 제안한다.
   - 보르도는 생장역~중심가와 가론 강변, 스트라스부르는 중앙역~그랑딜 및 트램 축이 자연스러운
     후보지만 exact 노선·도어 카피는 Claude/오너 별도 SPEC에서 확정해야 한다.

2. **P0 — 스탬프 보상 루프의 지역 격차를 별도 RFC로 해소**
   - 26도시 중 19도시가 앨범 대상이 아니다. `WORLD_NODES` 전용 앨범과 `REGIONAL_WORLD_NODES`
     `noStamp:true`의 정책 차이를 먼저 결정하고, EMEA/APAC를 무조건 활성화하지 말고 방문 기념의
     단일 의미·기존 66노드 카운트·저장 호환을 함께 설계한다.

3. **P1 — 서울·부산·코트다쥐르·레만호의 빈 이동 구간을 클러스터로 보강**
   - 임의 POI 증식 대신 기존 권역별로 1~2개의 학습/NPC/프롭 클러스터를 배치한다. 서울·부산은
     도어/NPC도 0이라 학습 상호작용을 우선하고, 코트다쥐르는 해안 도시 간 이동 장치, 레만호는
     기존 도선 TP 5를 중심으로 체류 이유를 강화한다.

4. **P1 — NPC를 일본 4도시 밖으로 확장**
   - 22도시의 NPC가 0이다. 상호·인물 재현 없이 역무원·시장 상인·카페 직원 같은 재사용 가능한
     역할을 언어권별로 제안하되, 도어와 같은 노드에 합쳐 상호작용 중복을 줄인다.

5. **P2 — 도어 없는 11도시를 언어 트랙 단위로 묶어 순차 보강**
   - 1차: 리옹·보르도·스트라스부르(완주 직후인 불어권 3도시), 2차: 브뤼셀·코트다쥐르,
     3차: 베이징과 호주 3도시, 4차: 서울·부산. 기존 트랙 챕터와 세트 간 비중복을 유지한다.

6. **P3 — 숫자만 맞추는 보강은 금지**
   - 파리·런던은 면적 밀도는 낮아도 POI 22/24, 도어 6, TRANSIT 3으로 시스템 다양성이 높다.
     몽생미셸·가와구치코의 특수 지형도 별도 장면·도선으로 보완된다. 동일 개수 목표보다 도시별
     이동 거리와 플레이 장치의 종류를 기준으로 검수한다.

## 비구현 경계

이 감사가 승인하는 것은 우선순위 논의뿐이다. 도어 카피·TRANSIT 노선·NPC·스탬프 정책·runtime·
registry·verifier·DB 변경은 각각 별도 SPEC과 독립 PR이 필요하다.
