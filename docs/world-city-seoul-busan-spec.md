# 서울·부산 도시 정밀맵 geo 생성 스펙 (오너 지시 2026-07-16 — "일본 도시들처럼")

Codex 킥오프용. 기존 4도시(후쿠오카·도쿄·오사카·교토) 파이프라인 재사용:
OSM/실지형 기반 grid + RLE + POI/역 스냅(보행·카디널 인접·단일 성분) + railways mask.
분업: **geo·스냅·테스트 하네스 = Codex / 검증 desc·파사드·프롭·NPC·e2e = Claude**(geo 도착 즉시 배선).

## 공통 계약 (기존과 동일)
- 20m/타일, CITY_TILE 팔레트, ENTRANCE/EXIT 세로회랑(같은 x열·북쪽·전 구간 보행 — e2e '위-직진' 계약)
- 마커(노드+역) Chebyshev ≥3 이격, POI 전부 보행+보행인접, BFS 전역 도달성
- 결정성: byte-identical 재생성, RLE 왕복 보존 테스트(대형이면 timeout 20s 명시 — tokyo/kyoto 선례)
- CITY_MAPS 레지스트리 등록, 전국맵(legacy 한일) 서울/부산 노드에 city 게이트 연결

## 한국 도시 스키마 노트 (일본과 다른 점)
- POI source: `{ id, nameKo, tile, lon, lat }` — **yomi 필드 없음**(일본어 요미는 일본 지명 전용).
  테스트 계약은 `node.name === source.nameKo` + `desc.includes(source.nameKo)`로 대체.
- contentLocale: 'ko'. 콘텐츠 스코프(KR/JP)에 부합 — 헌장 §1 위반 없음.
- 부산은 기존 **부산↔후쿠오카 페리 계약**과 접점: 부산항 국제여객터미널 POI가 페리 게이트
  착지와 정합해야 함(fukuoka 하카타항 국제터미널 패턴 미러).
