## 5. 신규 발견 문제

신규 발견은 근거가 확정된 두 Important만 기록한다. JA-R2-I01은 이전 평가가
놓친 기존 내용이고, JA-R2-I02는 교정 커밋 뒤 현행 자동 검사에서 새로 드러난
출고 회귀다.

### JA-R2-I01 — `〜になります`·`よろしかったでしょうか`의 형식 자체를 X 처리

`grammar/n2.js:63-73`은 알바 경어를 설명하면서
`〜になります(X)`·`よろしかった(X)`라고 표제화하고, 두 형식이 입에서
나오는 순간 감점된다고 단정한다. 이 문구는 이전 기준에도 있었으며
`git blame`은 `85cff836`을 가리킨다.

문제는 해당 장면의 부적절성과 형식 전체의 문법성을 구분하지 않는다는 점이다.
`コーヒーになります`가 단순 제시에서 `です/でございます`의 대용이면
교정 대상이지만, 같은 트랙의 `上手になります`처럼 실제 변화의 `なる`는
정상이다. `よろしかったでしょうか`도 지금 처음 묻는 주문 확인의 기본형으로
권장하지 않는 것과, 이미 선택·확인한 과거 내용을 다시 확인하는 용법은
구분해야 한다. 금지 표지를 각각 “변화 의미 없는 명사 제시”와 “현재 사항의
첫 확인”으로 범위화해야 한다. N2 비즈니스 출력 규칙을 형식 전체의 금지로
오해하게 하므로 Important다.

### JA-R2-I02 — 교정 반례가 한자 레벨 검사를 1오류에서 7오류로 증가시킴

`node scripts/check-content.mjs`의 일본어 한자 오류는 이전 평가의
`形(N3)` 1건에서 현행 7건으로 늘었다. 새 여섯 위치는 모두 C/I 수정에서
추가한 반례·기준명이다.

| 위치 | 새 검사 대상 | 유입 근거 |
|---|---|---|
| `grammar/n5.js:58` | `行` | JA-I01, `3005de68` / #607 |
| `grammar/n5.js:219` | `来` | JA-I02, `3005de68` / #607 |
| `grammar/n5.js:912` | `彼·事·実·知` | JA-I03, `3005de68` / #607 |
| `grammar/n4.js:736` | `遊` | JA-I05, `3005de68` / #607 |
| `grammar/n3.js:31-39,74` | `伺·針` | JA-C02, `014f8b51` / #603 |
| `grammar/n3.js:666` | `丈` | JA-I06, `3005de68` / #607 |

교정 내용의 언어 사실은 유효하지만, 레벨별 가나화 또는 `kanjiExempt` 정책을
함께 처리하지 않아 현행 출고 검사가 실패한다. 이 보고서는 report-only이므로
콘텐츠나 예외 목록을 수정하지 않는다.

그 밖에 변경부와 고위험 축을 재검수했으나 새 Critical을 확정할 근거는 찾지
못했다. 가능성만 있는 후보는 신규 지적으로 올리지 않았다.

## 6. 한국어 설명·예문과 챕터/팩 재판정

### 한국어 문체와 자연성

이전과 같은 정규식
`(?:습니다|합니다|됩니다|입니다)(?=[.!?\s]|$)`으로 객체를 재귀 순회하면
번역 `ko` 밖 설명 영역의 격식체는 97건에서 **1건**으로 줄었다. 남은 1건은
`grammar/n3.js:78` note의 “다나카라고 합니다.”다. #609의 대량 수리는
효과가 분명하므로 한국어 설명 축은 C에서 B로 올린다.

반면 이전 자연성 표본은 I12를 제외하면 대표 사례가 그대로다.

| 위치 | 현행 | 재판정 |
|---|---|---|
| `grammar/n5.js:824-825` | 역무원에게 `すみません、渋谷行き？` | 정중한 모범 출력으로는 `この電車は渋谷行きですか`가 안전 |

## 7. 종합 등급 변화와 남은 합격선

이전 D는 JA-C01·JA-C02라는 두 핵심 체계 오류가 직접 만들었다. 두 Critical은
수리됐고, 이전 Important도 **12 수리·부분 0·잔존 0**이다. 현행에는
**Critical 0, 신규 Important 2**가 있으며 이전의 자연성·문형·출처·콘텐츠
정책 백로그가 남는다. 종합 등급은 축의 산술 평균이 아니라 가장 위험한 학습
규칙과 출고 계약으로 정하므로 **D → C**가 타당하다.

다음 조건을 충족하면 B 이상을 다시 검토할 수 있다.

1. JA-R2-I01을 사용 장면별 규칙으로 범위화하고 N2 경어 예문을 재검수한다.
2. 새 한자 레벨 오류 6건과 기존 `形(N3)` 1건을 정책에 맞게 해소해
   `scripts/check-content.mjs`를 green으로 만든다.
3. 남은 자연성·한국어 표본과 설명 영역 격식체 1건을 문맥 검수한다.
4. N4·N3·N2 문형 목표와 레벨 간 중복의 교육적 역할을 정리한다.
5. 비공식 JLPT 목록·생성 예문의 출처와 검수 상태를 명시한다.
6. 상호·작품·캐릭터 재현을 저장소 정책에 맞게 일반화한다.
7. 수정 후 콘텐츠 lint, 한자·요미가나 검사, 전체 Vitest를 모두 green으로
   재검증한다.
