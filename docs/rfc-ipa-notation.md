# RFC — IPA·리에종 표기 정책 v1 (프랑스어 우선, 4트랙 일반화)

상태: 채택(2026-07-24). 프랑스어 전수 평가(#583 `docs/review-french-track.md`)가 요구한
"의무·선택·금지를 표시할 수 있는 IPA 정책"의 확정판. 신규 저작·수정분부터 즉시 적용,
기존 콘텐츠는 이행 라운드(R2~R4)로 소급한다.

## 1. 문제 (평가 #583 적발)

1. **선택 리에종의 유일 정답화** — `pas + 모음`의 [z] 연결(격식 변이)을 일상 예문의
   유일 발음처럼 IPA에 고정. 대표: `bunkei/a1.js:95,174,259-260`, `a2.js:55,253,653,880`,
   `b1.js:369-370,471,981`, `b2.js:419,865,1054`, `scene_travel.js:101-102,156-164` (FR-I03).
2. **IPA 필수 범위 계약 부재** — 장면 이야기 37개 발화 + A1 본편 16·A1 확장 5·A2 21개
   예문에 IPA 없음. "어떤 필드에 IPA가 필수인가"가 정의된 적이 없다.
3. **표기 스타일 혼재** — 리에종 연결을 `‿`로 표시하는 파일(a1_pronunciation)과 자음을
   그냥 붙여 쓰는 파일(scene_travel `[ʒə sɥiz alɛʁʒik]`)이 공존.

## 2. 정책

### P1. 리에종 3등급 표기 (프랑스어)

| 등급 | 표기 | 예 |
|---|---|---|
| **의무** (한정사+명사, 대명사+동사, 수사+명사, très 등 부사+형용사, 단음절 전치사+보어) | `‿`로 잇는다 | les amis [lez‿ami] · vous avez [vuz‿ave] · quatre-vingts euros [katʁəvɛ̃z‿øʁo] |
| **선택** (pas·복수명사·조동사 뒤 등) | **기본형은 무리에종**으로 적는다. 격식 변이는 note에만 | Je ne suis pas arrivé [pa aʁive] + note "격식체·낭독체에서는 [paz‿aʁive]로 이어요" |
| **금지** (et 뒤, 단수명사+형용사, h aspiré 앞 등) | 잇지 않는다. 위반은 **치명 등급 오류** | et alors [e alɔʁ] (✗ [et‿alɔʁ]) |

근거: 학습자의 기본 레지스터는 일상 회화다. 의무 리에종만 몸에 붙이고, 선택 리에종은
"들었을 때 아는" 수준으로 인지시킨다. 선택을 기본값으로 고정하면 학습자가 모든 위치에서
리에종하는 과잉 일반화가 생긴다(#583 지적).

### P2. IPA 필수 범위 계약 (v1.1 개정 — 2026-07-25)

- **필수 범위: A0~A2 예문·장면 이야기·발음 챕터 전수**(R3에서 소급 완료 — 79/79).
- **B1~C2 예문(기존 272건)은 소급하지 않는다** — 판단 근거: 해당 레벨 학습자는
  철자-발음 규칙을 이수한 상태라 예문 전수 전사의 학습 가치가 낮고, 표기 부담이
  본문 품질 라운드를 밀어낸다. 단 **발음이 비직관적인 항목(고유명사·외래어·묵음
  변칙)에는 선택적으로 부여**하며, 신규 저작 시 이 기준을 권장으로 적용한다.
- `story` 발화·대화문(fr): **권장** — R3 라운드에서 소급 보충(장면 챕터 우선).
- 한글 근사 병기(`[ipa] 한글`)는 A0~A1에서 권장, A2+는 IPA 단독 허용.
  한글 근사는 보조 바퀴임을 항상 전제한다(a0 r=[ʁ] '임시 근사' 서술 선례, #591).

### P3. 표기 세부

- `‿`는 **의무 리에종에만** 사용한다. 스타일은 `[lez‿ami]`(연결 자음 뒤 `‿`)로 통일.
- 엘리종은 표기 결과 그대로 적는다: l'ami [lami] — 별도 기호 없음.
- 앙셴망(enchaînement, 자음-모음 이어읽기)은 기호 없이 붙여 적는다(기존 관례 유지).

### P4. 검증

- lint 확장(후보, R4): ① fr 있는 example의 ipa 누락 **카운트 지표**(report-only)
  ② 금지 리에종 소수 하드 패턴(`et‿` 등) 에러화.
- 선택 리에종의 문맥 판단은 자동화 불가 — **표본 사실 검수 절차**(정본화 표준)로 커버.

## 4. 4트랙 일반화 원칙

발음 표기는 "**표기는 사전형, 실제 변이는 note**"를 공통 원칙으로 한다.

- **중국어**: 성조는 원조(사전형)로 표기하고 변조(3성 연속·경성 높이·不/一)는 설명·note로
  — #589 h1_pronunciation에서 확립된 방식을 트랙 표준으로 승격.
- **일본어**: 가나 표기가 1차, 로마자는 보조. 장음·촉음·박(モーラ) 감각은 설명으로.
- **영어**: IPA에 **강세 기호 필수**(1음절어 제외). 영/미 변이는 한 형(미국식)을 기본으로
  통일하고 필요 시 note — 트랙 감사(codex4/en-track-review) 결과 반영해 부록 확정.

## 5. 비고

- 이 정책은 콘텐츠 규약이며 스키마 변경을 동반하지 않는다(ipa는 기존 필드).
- 화자명 fr 필드 분리·slug 불일치는 별도 설계(진도 키 이행 필요)로 이 RFC 범위 밖.
