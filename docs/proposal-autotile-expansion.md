# V3 도로 오토타일 확산 사전검증

## 결론

측정 기준 26도시의 모든 런타임 도로 타일은 현행 N/E/S/W `0..15` 중 하나로
분류됐고, **모든 도시가 16개 mask를 전부 실제 소비**한다. 따라서 4방 연결 위상만
놓고 보면 누락된 17번째 mask는 없다. 리옹 파일럿의 별도 `inner`
(`mask=15` + 대각 4칸 roadLike)도 도쿄·서울에서 그대로 판정 가능하다.

다만 16종만으로는 다음 문맥을 표현하지 못한다.

- 대각선의 연속 방향: 1타일 사선은 corner 2종의 계단으로만 보이고, 대각-only
  접촉은 서로 연결되지 않는다.
- 횡단보도의 원래 축: `mask=15`/`inner`처럼 가로·세로가 동률이면 현행 코드는
  세로 지브라로 고정한다.
- `ROAD↔BRIDGE` 진입부, 철도 병행, 수변 쪽 연석·난간: roadLike 4방 mask에는
  지형 문맥이 없다.

따라서 판정은 **서울 조건부 GO → 도쿄 bridge/crosswalk 라이브 게이트 뒤 GO**다.
제품 코드 확산 자체의 위상 blocker는 없지만, 도쿄를 리옹과 같은 무검수 opt-in으로
취급하면 안 된다. 본 문서는 report-only이며 runtime·도시 wrapper/geo·registry·
verifier·테스트·DB를 수정하지 않는다.

## 기준과 측정 정의

- 측정 base: `0cbd40976ef8d10945020385043da35e9066073e`
- 대상: [`CITY_MANIFEST`](../src/components/world/cities/manifest.js)의 26도시
- 격자: 각 도시 `city.buildGrid()` 결과. EXIT를 덧씌운 뒤이므로 EXIT는 도로 수에서
  제외되며, 실제 `CityScene` 청크 베이킹 입력과 같다.
- roadLike: `ROAD | CROSSWALK | BRIDGE`
- 4방 mask: N/E/S/W = `1/2/4/8`
- 사선 비율: corner mask(`3|6|9|12`)가 cardinal 이웃의 보완 corner mask와
  이어지는 **1타일 staircase 타일 / roadLike**. 모든 corner 비율도 상한 진단값으로
  함께 제시한다.
- 대각-only 접촉: 대각 roadLike 쌍 사이의 두 직교 연결칸이 모두 roadLike가 아닌 쌍.
  이 값은 실제 같은 도로라는 source 방향 정보가 없으므로 곧바로 오류 수로 해석하지 않고,
  4방 mask만으로 판정할 수 없는 감사 후보 수로 쓴다.
- 광폭 내부: 현행 [`roadInteriorAt`](../src/components/world/CityScene.js)의
  `mask=15` + 대각 4칸 roadLike.
- 교차 밀도: `inner`를 제외한 T/십자(`popcount(mask) >= 3`) 후보를 4방 성분으로
  묶은 수 / roadLike 10,000타일. 실제 OSM 교차로 개수가 아니라 오토타일의
  교차 변형 군집 밀도다.
- 철도 병행: 직선축 roadLike의 같은 칸 또는 좌우/상하 1칸에 동일 축으로 계속되는
  railway mask가 있는 타일.
- 수변 인접: cardinal 이웃에 `WATER` 또는 `RIVER`가 있는 roadLike 타일.

현행 계약은 [`ROAD_AUTOTILE_VARIANTS`](../src/components/world/CityScene.js#L94-L109),
4방/inner 판정은
[`roadMask`·`roadInteriorAt`](../src/components/world/CityScene.js#L1151-L1167)에 있다.
철도는 도로 베이크 뒤 별도 `ct_rail_area`로 덮이므로
([`bakeChunk`](../src/components/world/CityScene.js#L1475-L1491)), 병행 철도는
추가 도로 위상이라기보다 합성 감사 대상이다.

### 리옹 대비

- 도쿄 roadLike는 리옹의 `4.989×`다. 사선 staircase는 `+0.037%p`로 거의
  같지만, inner는 `+4.116%p`다. 넓은 도로 때문에 교차 후보 성분 밀도는
  `-56.2%`이고, mask 15가 큰 면으로 뭉치는 경향이 강하다.
- 서울 roadLike는 리옹의 `8.996×`다. 사선 staircase `+0.142%p`,
  inner `+2.131%p`, 교차 후보 밀도 `+4.7%`로 리옹과 분포가 가깝다.
  다만 대각-only 접촉은 roadLike 1만당 `907.27쌍`으로 도쿄 `250.58`,
  리옹 `757.75`보다 많다.
- 수변 인접 비율은 도쿄 `2.07%`, 서울 `2.39%`, 리옹 `2.44%`로 밀도 자체는
  새 outlier가 아니다. 해안/강변 전용 외곽선이 없다는 시각 문맥 공백은 세 도시
  모두 공통이다.

## 16종 밖의 엣지 판정

| 엣지 | 도쿄 | 서울 | 리옹 | 판정 |
|---|---:|---:|---:|---|
| 4방 mask 실제 소비 | 16/16 | 16/16 | 16/16 | 4방 위상 누락 없음 |
| 사선 staircase | 9,864 | 18,642 | 1,943 | corner 교대로 연결되지만 사선 픽셀은 표현 못 함 |
| 대각-only 접촉 쌍 | 11,360 | 74,170 | 6,886 | 4방 mask만으로 동일 도로 여부 판정·연결 불가 |
| CROSSWALK inner | 10,408 (43.78%) | 6,049 (45.95%) | 2,621 (31.54%) | 동률 축이 모두 세로 지브라로 폴백하는 확대 리스크 |
| 철도 병행 | 44,165 | 35,733 | 5,528 | 후처리 overlay라 topology 추가 불필요, 합성 확인 필요 |
| 수변 인접 | 9,364 | 19,549 | 2,218 | 일반 비도로 edge만 사용, 수변 연석·난간 의미 없음 |
| BRIDGE / inner | 16,429 / 4,927 | 0 / 0 | 0 / 0 | 도쿄가 두 후보 중 유일한 대규모 bridge 실전 |
| ROAD↔BRIDGE 전이 면 | 10,658 | 0 | 0 | 양쪽을 roadLike로 합쳐 진입부 cap·난간 전이를 표현 못 함 |

### 해석

1. **철도 병행 도로는 새 4방 variant가 아니다.** 철도 mask가 지형 위에 별도
   반투명 overlay로 그려져 연결 위상은 유지된다. 다만 도쿄는 roadLike의 9.74%,
   서울은 4.37%가 병행 후보여서 파선·지브라와 철도 tint가 겹치는 화면을 확인해야 한다.
2. **해안·강변 도로도 4방 연결은 커버한다.** 현재는 물 쪽도 다른 비도로 변과 같은
   1px edge다. 수변 난간·연석이 필요하면 16종을 늘리는 대신
   `waterSideMask` 같은 독립 modifier/overlay가 맞다.
3. **사선은 topology와 외형을 분리해야 한다.** staircase는 corner mask로 끊기지
   않지만 계단 외형을 피할 수 없다. 대각-only 쌍은 source way 방향 없이는 같은 도로인지
   알 수 없다. 통계만 보고 자동 연결하면 인접한 별도 도로까지 합칠 수 있으므로 금지한다.
4. **도쿄 BRIDGE는 별도 게이트다.** 16종의 bridge surface는 모든 mask를 만들지만,
   리옹은 BRIDGE 0이라 대규모 `ROAD↔BRIDGE` 전이가 파일럿에서 검증되지 않았다.
5. **횡단보도 축은 mask만으로 부족하다.** 현행
   [`ct_crosswalk_autotile`](../src/components/world/CityScene.js#L279-L295)은
   가로·세로 연결 수가 같으면 한 축으로 고정한다. 서울·도쿄는 inner CROSSWALK 비율이
   리옹보다 각각 `+14.41%p`, `+12.24%p` 높다. 필요하면 `CROSSWALK` 자체 이웃축이나
   generator의 원래 way bearing을 별도 힌트로 써야 한다.

## 2차 확산 전 최소 게이트

- 서울: 일반/잠금 지구 각각 `inner CROSSWALK`, 철도 병행, 한강 수변,
  사선 staircase를 1개 이상 같은 줌에서 전후 캡처.
- 도쿄: 위 4종에 `BRIDGE inner`, `ROAD↔BRIDGE` 진입부, 수면 위 교량을 추가.
- 두 도시 모두 roadLike 전수에서 16 mask + inner 외 texture key 0,
  원래 연결 성분·충돌·EXIT 불변, 동일 입력 2회 render-key SHA 일치.
- 횡단보도 축이 육안으로 틀리면 도시 opt-in을 강행하지 않고 source bearing 또는
  crosswalk-only 이웃축 계약을 먼저 제안한다.
- 대각-only 접촉은 source way ID/방향 없이 자동 연결하거나 지형을 수정하지 않는다.
