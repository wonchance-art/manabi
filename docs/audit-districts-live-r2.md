# W2 — 지구제 개선 반영 라이브 감사 r2

## 판정

최신 `main`에서 V1·D5·D3·S6는 확인됐지만, W2 범위에 포함된 S7은 아직 `main`에 없다.
결함·개선 후보는 아래 3건이다.

| ID | 우선순위 | 후보 |
|---|---:|---|
| W2-R2-01 | P1 | S7 여행 수첩 다음 목표 1줄이 PR #451 draft에만 있고 최신 `main`에는 없어, D3·S6·S7 통합 상태를 라이브 판정할 수 없다. |
| W2-R2-02 | P2 | 코트다쥐르의 밝은 보도↔잠금 평지 경계는 V1 점묘가 생겼지만 16×16 타일 평균 명도 차가 여전히 작다. |
| W2-R2-03 | P3·dev DX | `?spawn=city:<id>`는 해당 오버월드 도시 스탬프를 심지 않는다. 미로그인 dev guest는 수첩의 도시 상세(D3·S6)를 곧바로 재현할 수 없다. |

런타임 진행 차단, 리옹 포장 회귀, 리옹·보르도 팻말 겹침은 발견하지 않았다. 이 문서는
report-only이며 런타임·도시 데이터·테스트·DB·API를 수정하지 않는다.

## 기준과 관찰 제한

- 착수 정본: `origin/main` `fbf448c15d38247b9a7aef862a87d7ca3ce80edf`.
- 환경: 공식 nvm Node `v22.23.1`,
  `NEXT_PUBLIC_WORLD_DEV_GUEST=1`, `npx next dev -p 3005`.
- 3005 라이브 응답:
  `city:lyon@239,139`, `city:bordeaux@315,191`,
  `city:strasbourg@110,257`, `city:cote-dazur@600,191` 모두 HTTP 200.
- 캡처 가능한 브라우저 backend가 없어 스크린샷을 만들지 못했다. 발주 허용대로 라이브 SSR
  DOM/텍스트(`로딩 중...`, `/world` 200)와 결정적 렌더 산출·프레젠테이션 문자열 관찰로
  대체했다. 따라서 “화면에서 따라 걷고 싶은가” 같은 주관 미감은 최종 판정하지 않는다.
- 첫 3005 기동은 기존 `.next`와 watcher의 `EMFILE` 때문에 `_not-found`만 컴파일했다. 기존
  캐시를 `/tmp`에 보존 이동하고 polling watcher로 재기동하자 4개 URL이 모두 200이 됐다.
  제품 라우트 결함으로 집계하지 않았다.
- `.env.local`은 존재 여부만 확인했고 읽거나 수정하지 않았다.

## 1. V1 이후 리옹 포장 — 주간/저녁/야간

리옹 mainRoute가 실제 소비하는 같은 ROAD 샘플에서 V1 합성 후 RGB와 시간대 조명
`0x18254a`를 적용해 명도 차를 비교했다.

| 시간대 | ROAD RGB | 포장 RGB | 포장−ROAD 지각 명도 |
|---|---|---|---:|
| 주간 | `[74,77,85]` | `[116,109,98]` | `32.754` |
| 저녁(α 0.10) | `[69,73,84]` | `[107,102,96]` | `29.686` |
| 야간(α 0.24) | `[62,67,82]` | `[94,92,92]` | `25.405` |

세 시간대 모두 V1 회귀 하한 `20`을 넘는다. 주간 대비 야간 신호가 약해지지만 하한 아래로
떨어지지 않아 추가 결함 후보로 올리지 않았다. 스크린샷 환경에서 주관 미감만 재확인하면 된다.

## 2. D5 경계 팻말 — 리옹·보르도 실배치

| 도시 | 결정 배치 | W1 경계 스폰 안의 가까운 실배치 | 앵커 겹침 |
|---|---:|---|---|
| 리옹 | 62 | 스폰 `(239,139)`에서 팻말 `(237,136)` | node·station·prop·route prop·discovery와 Chebyshev 2타일 이내 0 |
| 보르도 | 75 | 스폰 `(315,191)`에서 팻말 `(310,188)` | node·station·prop·route prop·discovery와 Chebyshev 2타일 이내 0 |

두 도시 모두 팻말은 개방 쪽 보행 타일에 있고, 잠금 타일과 4방향으로 맞닿으며, 다른 팻말과
Chebyshev 8타일 이상 떨어진다. `CityScene`의 실제 `sceneProps`에 합쳐져
`ct_prop_district_boundary_signpost`로 소비된다. 배치 누락·중첩 후보는 없다.

## 3. 잠금 톤 — 스트라스부르·코트다쥐르

W1 경계 좌표에서 개방 보도와 잠금 텍스처의 16×16 RGBA를 비교했다. `mean abs`는 같은 위치
픽셀의 지각 명도 차 평균이고, `≥20`은 차이가 20 이상인 픽셀 수다.

| 도시·경계 | V1 전 | 최신 main | 관찰 |
|---|---|---|---|
| 스트라스부르 `(110,257)` 개방 SIDEWALK ↔ `(109,257)` 잠금 ROAD | mean abs `13.08`, max `43.99`, ≥20 `64/256` | mean abs `14.91`, max `56.57`, ≥20 `72/256` | 도로 본체·점선·점묘가 모두 강해져 추가 후보 없음. |
| 코트다쥐르 `(600,191)` 개방 SIDEWALK ↔ `(599,191)` 잠금 SIDEWALK→guidebook land | mean abs `5.57`, max `7.06`, ≥20 `0/256` | mean abs `3.15`, max `35.37`, ≥20 `8/256` | 점묘 8픽셀은 분명해졌지만 나머지 248픽셀은 밝은 보도와 가깝고 평균 차는 오히려 감소. |

W2-R2-02의 최소 후속 후보는 코트다쥐르 같은 연속 평지 경계에서 종이 바탕 또는 이음새의
면적 대비를 올리는 것이다. 도로·수면·랜드마크 팔레트까지 다시 올릴 필요는 없다.

### W2-R2-01 — S7 통합 대기

S7 구현 `614426f32cf8ac2d6bece42f1a98cd095c2cbe40`은
`codex3/next-goal-line` PR #451의 draft 상태이고 이 감사 기준 SHA의 조상이 아니다.
따라서 현재 main은 D3 count와 S6 발견률을 각각 표시하며, S7의 카드당 “다음 목표 1줄” XOR
정리는 관찰 대상에 존재하지 않는다. #451 merge 뒤 같은 7도시 표를 다시 확인해야 한다.

### W2-R2-03 — dev guest 재현성

도시 상세는 `badge.has && cityDetailReady`일 때만 렌더된다. `?spawn=city:<id>`는 대상 도시맵을
선로드·진입할 뿐 오버월드 도시 스탬프를 지급하지 않는다. 미로그인 guest의 stamp GET도 빈 집합으로
폴백하므로, 직행 하니스만으로는 위 D3·S6 문자열을 DOM에서 볼 수 없다. 제품 저장 계약을 바꾸지 않는
dev-only stamp fixture 또는 `?stamps=` 감사 파라미터가 있으면 수첩 회귀를 독립 재현할 수 있다.

## 검증

- 타깃: 지구 렌더·mainRoute·D5·D3·S6 7파일 / 67테스트 PASS.
- D5 대표 단일 테스트: 1파일 / 5테스트 PASS, Vitest `--logHeapUsage` 최대 414 MB heap.
- 전체: ignored 복제본 `dist/d5-work`를 `/tmp`에 보존 이동한 뒤
  `set -o pipefail; npx vitest run --no-file-parallelism` —
  217파일 / 2,166테스트 PASS, 403.52초, exit 0. 복제본은 실행 직후 원위치 복원.
- 현재 샌드박스는 `/usr/bin/time -l`의 `sysctl kern.clockrate`와 `ps`를 차단해 RSS는
  실측할 수 없었다. 위 heap 수치만 메모리 증거로 사용한다.
