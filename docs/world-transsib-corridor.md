# 시베리아 횡단 연결선 MVP 계약

## 범위

- 씬 ID: `transsib-corridor`
- 노선: 블라디보스토크–모스크바, 주요 9개 정차역
- 출시 상태: `releaseEligible=false`
- 지역 콘텐츠: 없음. 지형·철도·승하차만 제공한다.
- DB 변경: 없음. 기존 `world_positions(scene, x, y)`만 사용한다.

## 상태 순서

1. 플랫폼에서 방향을 고른다.
2. 반대편 종착 플랫폼 좌표를 `/api/world/position`에 먼저 저장한다.
3. 저장 성공 뒤에만 대기·탑승 상태로 전환한다.
4. 대기·운행·차내 정차 중 `local:state.persistable=false`를 유지한다.
5. 중간역 정차 중 직접 하차하면 해당 플랫폼 좌표를 `persistable=true`로 전환한다.
6. 내리지 않으면 다음 구간으로 계속 운행한다.
7. 운행 중 이탈·로그아웃 또는 종착역 미하차는 2번에서 저장한 종착 플랫폼으로 복귀한다.

## 임시 시간표

- 배차 간격: 게임 시간 15분
- 정차 시간: 게임 시간 2분
- 구간 시간: `18, 22, 19, 16, 15, 18, 20, 16`분
- 상태: `provisional-owner-review`. 화면 연출과 밸런싱 전 오너가 확정한다.

## 안전 조건

- 종착 좌표 저장 실패 시 탑승하지 않는다.
- 서버는 `transsib-corridor`의 9개 플랫폼 타일만 저장하며 선로·차내 좌표는 거부한다.
- 차가 움직일 때 하차할 수 없다.
- 중간역에서 내리지 않아도 입력 잠금 없이 다음 구간으로 넘어간다.
- 씬 이름은 정확히 `transsib-corridor`만 허용하며 유사 문자열은 저장하지 않는다.
- 운행 재개 상태, 좌석, 현재 구간을 DB에 기록하지 않는다.

## 씬 구현

- `TranssibCorridorScene`은 지역 ①·②와 분리된 209×13 선형 씬이다.
- 9개 역마다 9×4 타일 플랫폼만 걸을 수 있고 플랫폼 사이 선로 보행은 막는다.
- 화면은 캐릭터를 따라가다가 탑승 후 열차를 따라가며, 정차 중에만 `A` 하차를 받는다.
- 출발역에서는 반대편 종착역을 고르고, 중간역에서는 양 종착 방향 중 하나를 고른다.
- 같은 씬의 다른 이용자는 MVP에서 표시하지 않는다. 장거리 이동 AOI 규칙 확정 뒤 별도 도입한다.
- 블라디보스토크 플랫폼은 `overworld:asia-pacific`의 `(1576, 442)` 게이트와 양방향 연결된다.
- 모스크바 플랫폼은 `overworld:emea`의 `(768, 253)` 게이트와 양방향 연결된다.
- 양방향 전환은 도착 씬의 정확한 플랫폼/게이트 좌표 저장 성공 뒤에만 실행한다.
- 로컬 검증은 월드 디버그 브리지의 `enterTranssib(stopId)`와 `enterRegion(regionId)`를 사용한다.
