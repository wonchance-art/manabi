## 4. 어느 렌즈도 보지 않은 것 — 진짜 새 위험 (완결성 비평)

6개 렌즈 전부가 놓친 사각지대. **"이 부분은 완료됐다"고 선언하기 전에 반드시 평가하라:**

1. **접근성(a11y)** — 플래시카드·타이핑·청취 문항의 키보드/스크린리더 흐름 미검증. 학습 '가능 여부' 자체를 좌우.
2. **복습 리마인더 / 장기 지속** — XP를 철거한 뒤 **무엇이 재방문을 유도하나?** due 알림(푸시/이메일) 존재 여부 불명. FSRS 앱에서 가장 치명적일 수 있는 공백.
3. **모바일/반응형** — 전 조사가 데스크톱 네비 기준. 실제 뷰포트 사용성 미검토.
4. **프라이버시/PII** — `review_events.detail`(오답 프롬프트·응답 jsonb)의 admin 조회 시 PII, 작문 텍스트의 제3자(Gemini/Groq) 위탁 고지. → **§3 admin RLS 정책을 열 때 반드시 함께 판단**(단순 RLS 추가로 충분한지, 익명화 필요한지).
5. **테스트 커버리지** — `fsrs`/`srs`/`grammarSrs`/`skillRung` 상태전이 로직에 회귀 안전망이 있나? rung 승급·EWMA·FSRS 재스케줄은 정교해서 회귀 없이 수정하려면 테스트가 전제.
6. **LLM 비용 상한** — 4개 AI 라우트에 rate limit 외 예산 가드 없음.
7. **온보딩 퍼널 실측(analytics)** — 이 진단 **전체가** '어느 화면이 사용자를 잃는지' 실측 없이 코드 구조로만 추론. UX 진단은 정성적 추정임을 명심 — 다음 큰 결정 전에 최소한의 퍼널 계측을 붙여라.

## 5. Fable의 종합 판단 (렌즈 교차)

- **퍼널이 먼저다.** 콘텐츠·메커니즘 개선이 의미를 가지려면 신규·FR/ZH 사용자가 애초에 v2 루프에 올바로 진입해야 한다. 그래서 이번 세션은 다른 무엇보다 **최상위 깔때기부터** 고쳤다. impact 5 후보 셋(퍼널·admin RLS·AI 오류교정 UI) 중 ROI가 압도적으로 확실한 것이 퍼널(effort 1)이다.
- **'2등 시민 FR/ZH' 실을 하나로 묶어라.** 온보딩 선택지 부재 · 레벨체계 결측(**ZH는 아예 없음**) · 추천 라벨 오표시 · 2차 보강 소스 편중 — 같은 병이 여러 렌즈에 흩어져 있다. 이를 **'언어 형평성' 단일 과제**로 다루면 처방이 일관된다.
- **'의도된 설계 vs 방치된 미완성' 기준을 문서화하라.** 렌즈마다 잣대가 달랐다(N1 축소는 봐주고 EN 빈곤은 문제삼음, tts는 봐주고 admin RLS는 최상위 유지). 제안 기준: **파일/플랜에 '의도'가 명시됐으면 스코프 선택, 아니면 부채.**
- **문서 부패가 구조적 문제다.** eval 문서가 3개월·110커밋 동안 미갱신 → XP 철거를 반영 못 해 '죽은 극찬'이 남았다. 오너십·주기를 정하지 않으면 **이 로드맵도 같은 운명.** → 규율 제안: **각 PR이 관련 전략문서 한 줄을 갱신**한다.
- **교차 검증이 필요한 불확실성 하나:** 어휘 저장 규약 이원화(surface vs base_form)가 사실이면, rung 계산이 전제하는 '`item_key`는 안정 식별자'가 흔들릴 수 있다 — 같은 단어가 다른 키로 흩어져 rung이 아예 안 쌓일 위험. **플랜 M3(규약 통일)를 서재·rung 작업의 공통 선행조건으로 취급하라.**
