# Anatomy Studio — 까다로운 학습자 시점 평가 및 전략 설계

> 평가일: 2026-03-30

## 총점: 6.2 / 10

---

## I. 항목별 평가

### 1. 핵심 학습 루프 (Core Learning Loop) — 7.5/10

**잘 한 점:**
- FSRS v4 알고리즘을 직접 구현한 것은 기술적으로 인상적. Anki 수준의 과학적 복습 스케줄링.
- 형태소 분석 → 단어 저장 → SRS 복습이라는 파이프라인이 명확하게 연결됨.
- 4단계 평가 (Again/Hard/Good/Easy) + 안정도/난이도 추적은 학습 연구에 기반한 설계.

**부족한 점:**
- **복습 세션이 빈약함.** 카드 뒤집기(앞면→뒷면) 하나뿐. Anki처럼 타이핑 모드, 리스닝 모드, 예문 컨텍스트 퀴즈 등 다양한 복습 형식이 없음.
- **학습 진행률 시각화가 약함.** 12주 히트맵과 기본 통계만 있고, "N3에서 N2까지 얼마나 왔는가"에 대한 레벨 진행도가 없음.
- **일일 학습 목표가 단어 5개 수집으로 고정.** 사용자가 목표를 커스터마이즈하거나 복습 목표를 별도로 설정할 수 없음.

---

### 2. 콘텐츠 생태계 (Content Ecosystem) — 5/10

**잘 한 점:**
- NHK Easy, Wikipedia, Wikinews 등 다중 소스에서 자동 콘텐츠 수집.
- 공개/비공개 자료 공유 시스템.
- 언어/난이도 필터링.

**치명적 부족:**
- **자체 큐레이션된 고품질 학습 콘텐츠가 0개.** Duolingo, LingQ, Busuu는 수천 개의 그레이디드 리딩 자료를 보유. 여기는 사용자가 직접 텍스트를 붙여넣거나 랜덤 Wikipedia 기사에 의존.
- **오디오/리스닝이 전혀 없음.** 언어 학습에서 리스닝은 필수. TTS조차 없음.
- **YouTube 자막 추출은 있지만 비디오 플레이어 통합이 없음.** 자막만 텍스트로 보여주면 학습 효과가 반감.
- **문법 학습이 즉석 AI 분석에만 의존.** 체계적인 문법 커리큘럼이 없음.

---

### 3. UX/UI 완성도 (Polish & Feel) — 5.5/10

**잘 한 점:**
- 다크/라이트 테마, 따뜻한 색감의 디자인 시스템이 일관적.
- 뷰어 커스터마이징(폰트 크기, 행간, 자간, 후리가나 토글)은 세심함.
- 반응형 모바일 대응.

**부족한 점:**
- **컴포넌트 라이브러리 없이 순수 CSS.** 일관성이 좋지만, 애니메이션/트랜지션/마이크로 인터랙션이 최소한. 메이저 앱 대비 "아마추어" 느낌이 남.
- **빈 상태(empty state) 처리가 텍스트 메시지 수준.** Duolingo 같은 앱은 빈 상태에서도 삽화/캐릭터/동기부여 요소를 넣음.
- **온보딩 후 첫 사용 경험이 가파름.** "자료를 직접 찾아서 붙여넣으세요"라는 시작점은 초보 학습자에게 진입 장벽.
- **500+ 줄짜리 모놀리식 뷰 컴포넌트들.** 유지보수 부채이자 새 기능 추가를 어렵게 만듦.

---

### 4. 게이미피케이션 & 동기부여 (Motivation) — 4/10

**있는 것:**
- 스트릭 카운터 + 마일스톤 (7/30/100일).
- 일일 단어 수집 목표 (프로그레스 링).

**완전히 없는 것:**
- XP/포인트 시스템 없음.
- 레벨업/뱃지/어치브먼트 없음.
- 리더보드 없음.
- 챌린지/미션 시스템 없음.
- 학습 완료 시 축하 애니메이션이 빈약함.

Duolingo가 언어 학습 1위인 이유의 50%가 게이미피케이션임. 이 영역이 거의 빈 캔버스.

---

### 5. 소셜 & 커뮤니티 (Social) — 5/10

**잘 한 점:**
- 포럼은 기본기가 잘 갖춰짐 (글/댓글/좋아요/멘션/알림).

**부족한 점:**
- **학습과 소셜이 연결되지 않음.** 같은 자료를 읽는 사람끼리 토론하거나, 단어장을 공유하거나, 서로 퀴즈를 내는 기능이 없음.
- **학습 파트너/그룹 스터디 없음.**
- **포럼이 범용 게시판이라 학습 맥락이 희석됨.**

---

### 6. 기술 인프라 & 확장성 (Tech) — 7/10

**잘 한 점:**
- Supabase RLS, FSRS 직접 구현, React Query 캐싱, N+1 쿼리 최적화 등 백엔드 설계 수준이 높음.
- AbortController, 배치 처리, 폴링 등 비동기 패턴을 잘 씀.

**부족한 점:**
- **TypeScript 없음.** 프로젝트 규모가 커지면 타입 안정성 문제가 불가피.
- **테스트 코드 0개.** FSRS 알고리즘 같은 핵심 로직에 테스트가 없으면 위험.
- **Rate limiting이 인메모리.** 서버 재시작 시 리셋.

---

### 7. 차별화 (Differentiation) — 6/10

"텍스트 해부"라는 컨셉은 매력적이고, 형태소 분석 + SRS 조합은 LingQ와 유사하면서도 AI 분석이라는 차별점이 있음. 하지만:
- LingQ는 이미 15년간 같은 컨셉을 갈고닦았고 수만 개의 콘텐츠를 보유.
- 현재 상태로는 "AI가 파싱해준다"는 것 외에 LingQ 대비 뚜렷한 우위가 부족.

## III. 전략 설계: "메이저와 싸우지 않되, 메이저가 못 하는 것을 한다"

### 포지셔닝 전략

> **"AI-Native Immersion Reader"** — 원문 몰입 읽기에 특화된 AI 학습 도구

Duolingo(초급 게이미피케이션)나 Anki(범용 SRS)와 정면 대결하지 말 것. 대신 **"중급 이상 학습자가 실제 콘텐츠를 읽으며 어휘를 늘리는"** 니치를 공략.

---

### Phase 1: 기초 체력 (4주) — "안 쓸 이유를 제거한다"

| 우선순위 | 작업 | 임팩트 | 난이도 |
|:---:|------|:---:|:---:|
| P0 | **TTS 통합** — Web Speech API로 문장/단어 발음 재생. 일본어/영어 모두 지원. 뷰어에서 문장 탭 시 음성 재생, 단어 카드에 발음 버튼 | ★★★★★ | 낮음 |
| P0 | **복습 모드 다양화** — 타이핑 테스트(의미→단어), 리스닝(TTS→의미 선택), 문맥 퀴즈(빈칸 채우기, source_sentence 활용) | ★★★★★ | 중간 |
| P0 | **레벨 진행도 시스템** — "N3까지 알아야 할 단어 X개 중 Y개 학습" 프로그레스 바. 급수별 어휘 마일스톤 | ★★★★ | 중간 |
| P1 | **스타터 콘텐츠 팩** — 레벨별 10개씩 사전 분석된 고품질 텍스트. N5 동화, N4 뉴스, N3 에세이, N2 소설 발췌, N1 사설 | ★★★★ | 중간 |
| P1 | **첫 사용 가이드 투어** — 가입 직후 샘플 텍스트로 분석→읽기→단어저장→복습 전 과정을 체험시키는 인터랙티브 워크스루 | ★★★★ | 중간 |

---

### Phase 2: 중독 루프 (4주) — "매일 돌아올 이유를 만든다"

| 우선순위 | 작업 | 임팩트 | 난이도 |
|:---:|------|:---:|:---:|
| P0 | **일일 미션 시스템** — "오늘의 3가지": (1) 복습 N개 완료, (2) 새 단어 M개 수집, (3) 자료 1편 읽기 완료. 완료 시 XP + 보상 | ★★★★★ | 중간 |
| P0 | **XP & 레벨 시스템** — 모든 학습 행위에 XP 부여. 단어 복습 10XP, 자료 완독 50XP, 연속 7일 보너스 100XP. 레벨업 애니메이션 | ★★★★★ | 중간 |
| P1 | **어치브먼트 뱃지** — "첫 100단어", "30일 연속", "N3 마스터", "1000XP" 등 30개+ 뱃지. 프로필에 표시 | ★★★★ | 낮음 |
| P1 | **주간 리포트** — 매주 월요일 학습 요약. 수집 단어, 복습 정확도, 읽은 자료, 레벨 변화. 이메일 또는 대시보드 | ★★★★ | 중간 |
| P2 | **커스텀 일일 목표** — 복습량, 수집량, 읽기량 각각 개별 설정 가능 | ★★★ | 낮음 |

### Phase 3: AI 차별화 (4주) — "이건 여기서만 된다"

| 우선순위 | 작업 | 임팩트 | 난이도 |

| P2 | **AI 작문 피드백** — 학습한 단어/문법으로 짧은 글 쓰기 → AI가 교정 + 개선 제안 | ★★★★ | 중간 |

### Phase 5: 기술 부채 해소 (병렬 진행)

| 작업 | 이유 |
|------|------|

## IV. 핵심 경쟁 전략 요약

```
┌─────────────────────────────────────────────────────┐
│          Anatomy Studio 포지셔닝 맵                   │
│                                                      │
│  초급자 ←───────────────────────────→ 고급자           │
│                                                      │
│  게이미피케이션        ┌──────────┐                    │
│  ↑   Duolingo          │ Anatomy  │  ← 목표 위치      │
│  │                     │ Studio   │                    │
│  │        Busuu        └──────────┘                    │
│  │                                                    │
│  │                          LingQ                     │
│  │                                                    │
│  ↓                                Anki               │
│  순수 도구                                             │
└─────────────────────────────────────────────────────┘
```

**한 줄 전략:** LingQ의 콘텐츠 밀도 + Duolingo의 중독성 + Anki의 과학적 SRS를 **AI로 자동화**하여, 중급 학습자가 "진짜 일본어/영어 콘텐츠"를 읽으면서 자연스럽게 실력이 느는 경험을 제공한다.

**핵심 차별점 3가지:**
1. **AI가 모든 텍스트를 즉시 학습 자료로 변환** (LingQ는 수동, Duolingo는 자체 콘텐츠만)
2. **FSRS v4 기반 과학적 복습** (Duolingo/Busuu 대비 압도적)
3. **무료** (LingQ $12.99/월, Busuu $13.99/월)

### 앱의 본질 (2026-04-14 기준 재정의)

> **"AI가 모든 텍스트를 형태소 단위로 해부하고, FSRS로 기억에 정착시키는 몰입형 읽기 학습 도구"**

핵심 파이프라인: `자료 발견 → AI 형태소 분석 → 단어 저장 → FSRS 간격 반복`

이 파이프라인은 잘 작동한다. 문제는 파이프라인 **바깥**에 있다:

1. **AI 오류 교정 부재** — 형태소 분석이 핵심인데 틀린 결과를 고칠 수 없음. 신뢰 문제.
2. **문법이 단어장과 분리** — 문법 노트가 복습 대상이 아님. 형태소의 "품사/구조" 절반이 활용 안 됨.
3. **쓰기/생산 연습 없음** — "읽고 외우기"에서 "쓸 수 있다"로의 전환점이 없음.
4. **게이미피케이션-학습 정렬 부족** — XP가 활동량을 측정하지 학습 깊이를 측정하지 않음.

### 다음 개선 방향 (우선순위순)

1. **AI 오류 교정 UI** — 사용자가 후리가나/의미/품사를 직접 수정하고 저장할 수 있는 인터페이스
2. **문법 노트 → SRS 연동** — 저장한 문법 패턴을 단어처럼 복습 대상으로 만들기
3. **쓰기 연습 모드** — 학습한 단어/문법으로 문장 만들기 → AI 교정
4. **XP 보상 체계 재설계** — 복습 정확도, 마스터 단어 수 등 질적 지표에 가중치
5. **JLPT/CEFR 기반 진행도** — 급수별 어휘 커버리지 시각화
