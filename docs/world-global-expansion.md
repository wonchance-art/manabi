# 월드 글로벌 확장 — 오버월드 2대 지역 설계 헌장 (2026-07-15)

> 오너 비전 + Claude×Codex 설계 라운드 2 수렴본. **두 지역의 preview v1 투영 판정 완료**, 나머지 LOCKED.
> 관련: docs/world-city-maps.md(도시 정밀맵 §6), docs/world-multiplayer.md.

## 1. 비전 (오너)

- 학습 월드를 전 세계로 확장 — **서로 전부 연결**된다는 전제 하에 핵심 지역부터 분할 시작.
- 언어 원칙: "그 나라에선 그 나라 언어" — 전 언어 커버 불가하니 **대표 언어 앵커** 기준, 어긋난 곳은
  콘텐츠 제외 가능. 단 **전역 로밍(걸어다니기)은 포기하지 않는다**.
- 현실 정보 융합: 노드에 실제 명소·맛집·문화 정보를 담는다(규슈 패턴의 전 세계 확장).
- **콘텐츠 스코프(오너 확정 2026-07-15, 개정 2026-07-16)**: 기본은 **한국·일본** — 타 지역은
  지형+이동수단. **개정: 유럽 1차 웨이브(그랑파리·몽생미셸·코트다쥐르)는 관광 desc +
  프랑스어 학습 콘텐츠 동시 진행**(오너: "프랑스어 학습 바로 가능하게 함께 진행").
  2축 구조 첫 실전: local=fr·reading=한글 발음 표기·gloss=ko.
- 티어 구조: **오버월드(명목 4.5km/타일 · 성긴 지형·로밍) + 정밀 도시(표준 20m/타일,
  제한 보행권 예외 4m/타일 · 문화 코어)** 2계층. 도시 축척 허용값은 `docs/world-city-maps.md §6.1`을 따른다.
  4.5km 는 **명목 스케일** — 교통 시간은 노선 시간표로 결정(축척으로 환산하지 않음).

## 2. 지역 정의

### ① 아시아-태평양 (현 한일맵의 확장·흡수)
- 경계(오너): 서=파키스탄·중국·몽골 국경 라인(~60°E), 북=시베리아 횡단철도 여유(~61°N),
  동·남=뉴질랜드(180°E/47°S).
- 규모(**preview v1 확정치**): 등장방형 표준위도 27.5°N 기준 **2,631×2,669 = 7,022,139타일**
  (256청크 11×11=121개). 정확한 타일 수는 content manifest의 `width`·`height`를 기준으로 한다.
- **빈 태평양/인도양 미포함** — 괌 등 원양 도서는 view-only(표면 표시·충돌 차단), 항공/배 게이트로만
  방문·통과 시 조망. 빈 바다는 단일 바다색.

### ② 유럽-지중해-중동-러시아 핵심
- 경계(오너): 유럽 전역 + 북아프리카(고대 로마 영토) + 중동 종교 요지(이스라엘 등) + 터키 +
  러시아 핵심 도시(모스크바 — 맵 동측 중간). bbox ≈ 11°W~50°E × 20~66°N.
- 규모(**preview v1 확정치**): 등장방형 표준위도 50.25°N 기준
  **964×1,137 = 1,096,068타일** (256청크 4×5=20개).

### 교통 UX 원칙 — "직접 하차(ride & disembark)" (오너 확정 2026-07-15)
- 장거리 노선형 교통(횡단철도·페리·항공 등 신규 교통)은 **목적지 선택→자동 도착이 아니다**.
  플레이어는 **이동수단에 탑승해 기다리고**, 역/정류장 **도착 시 직접 내린다**. 내리지 않으면
  다음 정차지로 계속, **미하차·로그아웃 시 종착역 도착**.
- 이 원칙으로 여정 영속이 단순화된다 — **무스키마 계약**(corridor 설계에서 Codex 확정):
  · 영속 행은 transit 상태가 아니라 **재접속 스폰 목표인 종착 좌표만** 표현한다.
  · **탑승 확정 전에 종착 좌표 저장을 완료**하고, 운행 중 heartbeat/pagehide 좌표 저장은
    persistable:false 또는 동등한 서버 규칙으로 억제해 종착 좌표를 보존한다.
  · 종착 scene 식별자를 기존 whitelist(normalizePosition·isPersistablePosition)에 추가한다 —
    코드 계약 확장이며 DB 스키마 변경 아님.
  · 운행 재개·transit 상태 자체를 영속해야 하는 요구가 생기면 그때 신규 컬럼/테이블 —
    오너 승인 항목으로 회부.
- 기존 도시 전철(행선지 선택→즉시 이동)은 도시 내 단거리 UX 로 당분간 유지 — ride 모드로의
  개편 여부는 후속 오너 결정.

### 연결
- 지역 = **별도 Phaser 씬 + 게이트**(항공 허브). 전 지구 단일 좌표계 재편은 하지 않음.
- ①↔② = **transsib-corridor 교통 전용 씬**(블라디보스토크~모스크바 저밀도 선형 맵 + 주요 정차역,
  열차 밖 자유 로밍 제한). ② 동측 확장으로 겹치기 안(기각 — 중복 지형·경계 동기화 비용).
- ② 내부 철도 v1 은 런던–파리–베를린–빈 중심축, 파리–마드리드 및 빈–로마·이스탄불 분기형
  연결로 시작한다. 이는 시간표가 아닌 **직접 연결 토폴로지**만 확정하며, 모든 구간 시간·정차 간격은
  오너 결정 전까지 런타임 계약에 넣지 않는다. 탑승 전 선택 종착역 좌표 저장, 운행 중 좌표 저장 금지,
  중간 정차 직접 하차, 미하차 다음 구간 진행, 재접속 종착역 폴백은 §2 무스키마 계약을 그대로 따른다.
- 실제 철도 선형 감사에서 대륙 6허브는 동일 그래프이나 런던은 Natural Earth 원본상 별도 컴포넌트다.
  런던–파리 실선 추종은 영불해협 철도 터널의 버전 고정 수동 오버라이드 또는 별도 터널 게이트 결정 전
  금지하며, 직선 보간으로 결손을 숨기지 않는다.
- 장기: 아메리카 등 추가 지역도 같은 계약으로 이어 전 세계 그래프화.

## 3. LOCKED 설계 결정 (Claude×Codex 합의 — 재론 불요)

1. **소스**: Natural Earth 10m 주 소스(퍼블릭 도메인). GSHHG 는 해안 QA/소도서 보충만(LGPL·2.3.7
   최종본 — 채택 시 버전·라이선스 명기). 고도 ETOPO 2022(출시 산출물에 네트워크 폴백 금지).
2. **레이어 계약**: 표면(육지·해안·평야·산지·강·호수) / 충돌(1bit) / **view-only(독립 1bit 마스크
   — 표면엔 표시·충돌은 차단)** / **경계·교통 희소 오버레이**(국경·철도·항로 — 패킹 격자 밖 별도
   희소 데이터). **수동 오버라이드는 렌더 레이어가 아니라** 각 파생 레이어에 적용되는 버전 고정
   입력/패치(공항·항만·교량·게이트 착지점 — 별도 버전·해시, §3-9 manifest 에 포함).
3. **포맷**: 256×256 청크, **표면 4bit+충돌 1bit+view-only 1bit** 패킹(§3-2 의 세 격자 레이어와
   1:1 — 희소 오버레이는 패킹 밖). 단일 JS 문자열 번들 폐기 — 진입+카메라 주변 청크만 정적
   바이너리 로드.
4. **선행 공사**: 오버월드 청크 런타임 완성 전 ① 본생성 금지. (현 WorldScene 은 전체 격자를
   Phaser Tilemap 으로 변환 — 대형화 불가 구조. CityScene 청크 RT 경험 재사용 + 비트패킹·충돌·
   노드 인덱스 신설.)
5. **래스터 규칙**: 타일 4×4 supersampling. 소도서는 전역 임계 제거 금지(면적 임계+whitelist,
   원양은 view-only 전환). 강은 scale rank 상위만. 국경은 충돌과 완전 분리.
6. **마이그레이션**(기존 한일맵): 노드 ID 불변 · 전 노드에 lon/lat+regionId 백필 · legacy 한일 씬과
   신규 ① 씬 **이중 운용** · 관리자 뷰어 겹쳐 비교 · 전 게이트/착지/도시 EXIT 검증 후 기본 씬 전환.
   수동 배치 노드는 arrivalOffset/착지 오버라이드 지원.
7. **민감지역 정책**(② 필수): de facto 경계(NE 기준) + 분쟁·휴전·미확정선은 동일한 중립 점선 +
   분쟁지 국가별 채색 금지 + 수도·귀속 단정 금지 + 지명 내부 ID/로케일별 표시명 분리 + 경계 데이터
   고정 버전·changelog + UN식 고지문("지도상의 경계·명칭·표시는 특정 지역의 법적 지위나 경계 주장에
   대한 승인 또는 지지를 의미하지 않습니다").
8. **언어 앵커**: 국가 폴리곤에서 언어 자동 추론 금지 — `defaultContentLocale`(국가) +
   `contentLocale`(노드) + `contentEligible` 플래그. 다언어 지역은 복수 앵커.
9. **결정성 계약**(§6.4 오버월드판): 입력 manifest(URL·버전·SHA-256·라이선스) · 투영 ID·파라미터 ·
   bbox·표본 위치·반올림·피처 정렬·supersampling 위치·레이어 우선순위 고정 · 동일 입력→전 파일·청크
   SHA byte-identical · degraded 시 출시 미기록 · 인접 청크 경계 일치(철도·강·해안 연속) · 전역
   content hash manifest.
10. **멀티**: "화면에 보이는 사람만 동기화"(오너 확정). 현 단일 채널 유지, 대량 접속 시 셀 단위 AOI
    구독으로 승격(후순위 백로그 — 셀 크기·hysteresis 는 그때 Codex 설계).
11. **실행 순서**: 투영 스파이크 → 한일 범위 **수직 슬라이스**(새 파이프라인+청크 런타임) → ① 지형
    → 노드 마이그레이션(이중 검증) → ① 기본 씬 전환 → ② 지형(§3-7 정책 확정 전 비공개) →
    transsib-corridor → 콘텐츠·언어 앵커 점진(**§1 스코프: KR/JP 만** — 타 지역은 지형+교통까지).
    ② 지형 생성은 수직 슬라이스 후 ①과 병렬 가능.

## 4. LOCKED — preview v1 투영 판정

- 두 지역 모두 **등장방형(equirectangular) + screen-axis**를 사용한다. 화면 위=북쪽, 오른쪽=동쪽을
  유지하며 D-pad·미니맵·직선 교통선의 방위 의미를 일치시킨다.
- ① 아시아-태평양: 중심경선 125°E, 기준위도·표준위도 27.5°N,
  `apac-equirectangular-screen-axis-v1`.
- ② 유럽-지중해-중동-러시아 핵심: 중심경선 25°E, 기준위도 45°N, 표준위도 50.25°N,
  `emea-equirectangular-screen-axis-v1`. 최초 45°N 가설은 45–55°N을 0.25° 간격으로 재평가해 교체했다.
- ② 재평가 근거는 `docs/generated/world-projection-emea-sweep/report.md`와 같은 디렉터리의 결정적
  `results.json`·`content-manifest.json`이다. 실제 surface·terrain·playability·boundary·transport
  preview manifest도 모두 같은 투영 ID와 파라미터를 기록한다.
- Sinusoidal·LAEA와 geo-axis는 비교군으로만 남긴다. 지역별 상이 투영은 여전히 아키텍처상 허용하지만,
  현재 두 preview v1은 같은 투영 계열과 조작축을 선택했다.
- **투영 확정은 공개 출시 승인이 아니다.** 투영 확정 뒤 별도 출시 게이트를 통과한 EMEA 산출물은
  `releaseEligible=true`, APAC의 기존 생성 산출물은 `releaseEligible=false`를 유지한다. 런타임 지역
  공개 상태와 생성 산출물 상태는 자산 참조에서 각각 명시한다.

## 5. 오너 결정 현황

- ✅ **여정 영속(확정 2026-07-15)**: 미하차·로그아웃 = **종착역 도착**. "직접 하차" 원칙(§2)에 따라

  컬럼/테이블이 필요해질 때만 오너 승인 회부.
- ⏳ 열차 시간 파라미터 — 정차 간격·구간 시간은 corridor 설계 시 Codex 제안 → 오너 확정.
- ⏳ 30~45 엔지니어링일 아크 수용(수직 슬라이스부터 단계 출하; KR/JP 콘텐츠 저작과 민감지역
  편집 검수는 별도 지속 작업).
- ✅ **APAC 공개·기본 스폰(확정 2026-07-16, #134)**: APAC 일반 공개, 신규·미저장 계정은
  **서울 default 스폰(임시)**. 아래 모어 기반 시작이 준비되면 대체.
- ✅ **모어(母語) 기반 시작 지역(확정 2026-07-16, 구현은 플레이 언어 준비 후)**:
  한국인은 한국에서, 일본인은 일본에서 시작 — 상대국으로 넘어가면 서로 만나는 환경.
  국가·언어가 추가될수록 복합 양상. **도항 게이트**: 기초 상대어(일본어/한국어) 과정 수료 후
  국경을 넘을 수 있음 — 원활한 소통을 위한 최소 장벽(철회된 "지역별 레벨 밴드"와 구분:
  지역마다 차등이 아니라 국경 1회 게이트). **학습→여행비**: 배울수록 자금이 쌓여 여행비를
  마련하는 루프 — 경제 RFC 기둥①(학습 기본 수입)과 접속.
- 📦 **두바이 경유 구상(오너 동기 확인 2026-07-16, 백로그 저장)**: 목적은 중동 지역이 아니라
  **환승 경험** — 인천→유럽 항공에 흔한 두바이 경유의 재현.
  · **1단계(권장·저비용)**: 공항 스토리 씬 문법의 "두바이 환승 라운지" 소형 실내 씬 + 장거리
    항공 노선에 경유지 스톱. "직접 하차" 원칙 그대로(내리면 라운지, 안 내리면 최종 목적지) —
    지형·민감지역 이슈 없음.
