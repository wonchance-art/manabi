## 1. 종합 재판정: C 유지

R2 종합 C를 **C로 유지**한다. 다만 정체가 아니라 축별 개선이 확인된 C다. PR
#616에서 EN-I02의 `vsKo` 절대화와 R2-I01의 B1 고정 확률표가 수리됐고, PR
#636에서 EN-I10 구절 IPA 10/10이 표제 구절 전체 전사로 바뀌었다. 설명 필드의
비해요체 계수도 4→1건이며, 남은 한 건은 금지 의미를 보여 주는 직접 인용이다.
이에 따라 언어 정확성은 C→B, 한국어 설명은 B→A로 올린다.

그러나 R2와 같은 합격 조건으로 보면 Important가 모두 닫히지는 않았다.
`scene_travel.js`의 OT 여행 이야기 발화는 여전히 IPA 0/48이고, 같은 OT 장면
`scene_emergency.js`는 17/17이어서 EN-I11의 구조 불일치가 남는다. PR #630은
EN-I12의 `draft` 슬러그 7건과 진도 별칭 이행을 완료했지만,
`DRAFT_UNWIRED` 상태 7/7을 공개 레지스트리 경계에서 제거하는 상태 모순은 바꾸지
않았다. 따라서 EN-I12는 잔존에서 부분으로 개선됐지만 종결은 아니다.

현재 Critical은 0이고, R2의 언어 규칙·구절 IPA 교정은 모두 닫혔다. 그럼에도
발음·IPA와 커리큘럼 정합 축에 동일 루브릭의 Important가 남으므로, “경미한 교정만
남은 B”로 올릴 근거는 아직 부족하다.

등급 기준은 R2와 동일하다.

| 등급 | 기준 |
|---|---|
| A | 정확성·구조·안전 계약을 충족해 현행 그대로 배포 가능 |
| B | 핵심 학습을 왜곡하지 않는 경미한 문체·자연성 교정 후 배포 가능 |
| C | 치명 오류는 없으나 언어 규칙·IPA·공개 상태의 중요 교정 필요 |
| D | 핵심 규칙 또는 건강·안전 의미를 재검수하기 전 배포 보류 |
| E | 여러 축의 체계적 치명 오류로 부분 수리보다 재설계가 우선 |
| F | 학습 자료로 사용할 수 없거나 안전하게 복구할 기준선이 없음 |

## 2. 범위와 재평가 방법

PR #616~637의 merge commit·변경 파일·PR 본문을 대조했다. 22개 merge commit
전부가 현행 HEAD의 조상인지도 확인했다.

## 5. R2 잔여·이월·신규 항목 전 건의 종결 여부

R2에서 C 상한을 직접 만들었거나 R2에서 새로 확인한 여섯 항목을 별도로 모으면
다음과 같다.

| 항목 | R2 | R3 | 판정 근거 |
|---|---|---|---|
| EN-I02 `vsKo` 절대화 | 부분 | **수리됨** | #616이 “기본적으로”와 본문 예외 참조를 추가 |
| EN-I10 C2 구절 IPA | 잔존 | **수리됨** | #636이 부분 전사 10건을 표제 구절 전체 전사로 교체 |
| EN-I11 여행 story IPA | 잔존 | **잔존** | 현행 실측 0/48, 동일 OT 긴급 story는 17/17 |
| EN-I12 draft slug·공개 상태 | 잔존 | **부분** | #630으로 slug 7/7·alias 이행 완료, status 공개 모순 7/7 유지 |
| R2-I01 B1 고정 확률표 | 신규 Important | **수리됨** | #616이 숫자를 감 잡기용 예시로 제한하고 화자·문맥·억양 변이를 명시 |
| R2-M01 비해요체 4건 | 신규 Minor | **수리됨** | #616이 산문 3건 교정; 남은 1건은 `a2.js`의 “내면 안 됩니다” 직접 인용 |

### #616 — EN-I02와 B1 확률표

`grammar/a1.js:453`은 이제 진행형을 “기본적으로” 변화 중일 때 쓴다고 낮추고,
바로 앞 본문의 일시적 뉘앙스 예외를 다시 가리킨다. 섹션 본문과 마지막 요약의
상충이 없어졌으므로 EN-I02를 종결한다.

`grammar/b1.js:403-415`의 숫자 자체는 남아 있지만 표 caption이 이를 “감 잡기용
예시”로 제한하고 확신도가 화자·문맥·억양에 따라 달라진다고 명시한다. 이는 R2가
제시한 C1과 같은 단서라는 합격 조건을 충족한다. 숫자를 사전 의미값으로 가르치는
문제는 닫혔다.

### #636 — EN-I10 구절 IPA

`vocab/c2.js:51-60`의 10개 `ipa`는 모두 단어 하나가 아니라 표제 구절 전체를
덮는다. 예를 들어 `The elephant in the room`은
`[ði ˈelɪfənt ɪn ðə ˈruːm]`으로 바뀌었다. 필드 존재만 확인하지 않고 10개
표제어와 전사 범위를 대조했으며, R2의 합격 조건을 충족한다.

### #630 — EN-I12 slug 이행과 남은 상태 모순

확장 7개 slug에는 더 이상 `draft`가 없고, `storageSchema.js:27-33`에는 구→신
별칭 7건이 등록돼 진도 키 이행 경로가 있다. 이 부분은 종결이다.

반면 `grammar/expansion.js:8,16,85,146,207,268,329,390`의
`DRAFT_UNWIRED`는 그대로이고, `index.js:32-34`는 status를 제거해 7개 챕터를
공개 registry에 넣는다. 소스 머리말은 정본·검수 완료라고 설명하므로 콘텐츠 승인
의도는 읽히지만, R2가 지적한 기계 판독 상태와 공개 소비의 모순은 현행 데이터에서
없어지지 않았다. 같은 합격 조건을 적용해 EN-I12를 부분으로 둔다.

### EN-I11 — 공유 인프라 merge로는 닫히지 않음

#632의 dialogue 스키마·렌더와 #634의 IPA 정책 문서는 merge됐지만,
`scene_travel.js`의 story 객체는 이미 `{ speaker, en, ko }` 구조였고 IPA 값은
추가되지 않았다. 명시적 TTS 대체 계약도 영어 콘텐츠·RFC에서 확인되지 않았다.
따라서 실제 값 0/48을 우선해 잔존으로 판정한다.

## 6. 신규 발견

**새로 확정할 Critical·Important·Minor는 없다.** R2 이후 영어 diff는 기존
지적의 교정 23행으로 한정되고, 전수 객체 순회·R2 항목 재검수에서 별도 확정 근거가
나오지 않았다.

이 문장은 영어 트랙에 오류가 절대 없다는 선언이 아니다. 이번 R3의 동일 루브릭,
전체 필드 검사, R2 이후 diff에서 새 ID를 부여할 만큼 재현 가능한 근거가 없었다는
뜻이다. EN-I11·I12는 신규가 아니라 R2 이월 항목으로 계속 추적한다.

| 예문 자연성 | B- | **B-** | R2의 번역투·placeholder 표본을 바꾼 영어 diff가 없음 |
| 범위·분량 | A | **A** | 68챕터·585문형·1,382어휘와 필수 필드 유지 |

언어 정확성과 한국어 설명의 상승은 실질적이다. 다만 종합은 축의 산술 평균이
아니라 배포를 제한하는 최저 핵심 계약으로 정한다. 발음·IPA와 커리큘럼 정합에
Important가 남으므로 종합은 **C → C**다.

## 8. 종합 변화 근거와 다음 게이트

R2 당시 C의 원인은 EN-I02, B1 확률표, EN-I10·I11·I12였다. R3에서는 EN-I02,
B1 확률표, EN-I10이 닫혔고 EN-I12의 slug 절반이 닫혔다. 따라서 남은 게이트는
다음 두 개로 좁혀졌다.

1. **EN-I11**: 여행 story 발화 48개에 같은 OT 긴급 장면 수준의 IPA를 제공하거나,
   해당 필드의 발음 지원을 대체하는 명시적 소비 계약을 확정한다.
2. **EN-I12**: 공개 정본 7개에서 `DRAFT_UNWIRED` provenance를 정합한 상태로
   바꾸거나, draft 상태라면 공개 registry에서 제외한다. slug·진도 alias는 이미
   끝났으므로 다시 rename하지 않는다.

두 항목이 닫히면 발음·IPA와 커리큘럼 정합의 B 이상을 다시 검토할 수 있다.
예문 자연성 B-의 번역투·placeholder 표본은 별도 경미 교정 라운드다.
