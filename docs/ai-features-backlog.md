# AI 기능 백로그 — 런타임 학습 보조 (보류 중)

> 2026-06-12 논의 저장. **아직 구현하지 않음.**
> 원칙: 런타임(사용자 요청마다 호출되는) AI는 저렴한 모델로. Fable 5 같은 최상위 모델은
> 빌드 타임 콘텐츠 생성(Claude Code 세션·배치)에만 쓴다.

## 모델 배정 원칙

| 용도 | 모델 | 이유 |
|---|---|---|
| 실시간·짧은 응답 (인라인 튜터, 예문 변형) | `claude-haiku-4-5` | 1초 내 응답, $1/$5 per MTok |
| 채점·구조화 출력 (작문 첨삭, 크로스링크, 리포트) | `claude-sonnet-4-6` | 품질·비용 균형, structured outputs |
| 비실시간 대량 작업 (드릴 생성, 주간 리포트) | Batches API (50% 할인) | 야간 배치, 지연 무관 |
| 콘텐츠 제작 (챕터·문형·연재물 집필) | 빌드 타임 — Claude Code 세션 | 런타임 비용 0, 결과물은 정적 자산 |

## 기능 목록 (우선순위순)

### P1 — 작문 첨삭 채점기 (퀴즈 produce 단계 내장)
- 사용자가 쓴 문장을 채점: 통과 여부, 교정문, "한국어식 사고가 드러난 지점" 한 줄, 관련 문형 사전 슬러그.
- `output_config.format`(json_schema)으로 `{pass, corrected, note, relatedPatterns[]}` 강제.
- UI: 새 화면 없음. 퀴즈 verdict 패널에 첨삭 카드 1장.
- 모델: Sonnet 4.6, effort 불필요(짧은 작업).

### P2 — 인라인 "왜?" 튜터
- 챕터 예문 옆 `?` 버튼 → 예문+챕터 컨텍스트로 2~3문장 답변. 챗봇 탭 만들지 않기.
- 모델: Haiku 4.5 스트리밍. 챕터 본문은 프롬프트 캐싱(stable prefix).

### P3 — 자료 ↔ 강의 크로스링크
- 자료 분석 시 문장 속 문형을 문형 사전 슬러그로 매핑 → 자료에서 챕터로, 챕터에서 자료로.
- 모델: Sonnet 4.6 배치. 문형 사전 인덱스(pattern+slug 목록)를 캐시된 프리픽스로.

### P4 — 맞춤 드릴 + 주간 리포트
- 퀴즈 오답·미통과 기록 기반 개인 드릴 세트(야간 배치), 주 1회 학습 리포트 홈 카드.
- 모델: Sonnet 4.6 + Batches.

### P5 — 시나리오 롤플레이
- 카페 주문·길 묻기 등 레벨 어휘 제약 대화, 종료 시 교정 요약.
- 모델: Haiku 4.5 (대화) + Sonnet 4.6 (요약). 마지막 순위.

## 공통 가드레일
- `stop_reason` 확인 후 content 읽기 (refusal 분기).
- 사용자별 일일 쿼터 테이블 (Supabase, RLS).
- 같은 입력 재채점 금지 — 결과 저장 후 재사용.
- API 키는 Vercel 환경변수, 서버 라우트에서만 호출.
