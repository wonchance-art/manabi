# 오버월드 지역 스트리밍 런타임

## 범위

| 지역 | 씬 ID | 크기 | 횡단철도 게이트 |
|---|---|---:|---|
| 아시아·태평양 | `overworld:asia-pacific` | 2631×2669 | 블라디보스토크 `(1576, 442)` |
| 유럽·지중해·중동 | `overworld:emea` | 964×1137 | 모스크바 `(768, 253)` |

- 두 지역 모두 `releaseEligible=false`다.
- 체크인된 playability preview `.owc`를 256×256 저장 청크 단위로 읽는다.
- 화면에는 카메라 주변 32×32 렌더 페이지만 유지하며 기존 오버월드 LRU 상한을 적용한다.
- 표면·충돌·view-only 판정은 같은 청크에서 읽는다. 바다·충돌·view-only 타일은 이동할 수 없다.

## 교통 노드 인덱스

- 각 지역의 횡단철도 게이트는 지역 레지스트리의 하드코딩 좌표만 신뢰하지 않고 별도
  `transport-nodes-preview-v1` content manifest와 `nodes/<cx>/<cy>.json`에서 읽는다.
- content manifest에 없는 청크는 네트워크 요청하지 않는다. 문서의 노드는 전역 타일 좌표가 소속된
  256타일 청크와 일치해야 한다.
- 씬 시작 시 레지스트리의 ID·표시명·corridor stop·투영 타일과 인덱스가 모두 일치해야 게이트를 표시하고
  입력을 연다. 누락·drift·보행 불가 타일은 지역 진입을 실패 처리한다.
- 현재 체크인된 노드는 기존의 블라디보스토크·모스크바 횡단철도 게이트뿐이다. 신규 공항·항구·역과
  시간표·운임은 이 인덱스 포맷 위에서 후속 manifest로 추가한다.

## 저장과 복귀

1. 지역 안 보행 좌표는 기존 `world_positions(scene, x, y)`에 저장한다.
2. 재접속 시 등록된 씬과 지역 범위 안 좌표만 복원한다.
3. 저장 타일이 실제로 보행 불가하면 해당 지역의 횡단철도 게이트로 폴백한다.
4. 지역에서 횡단철도로 이동할 때 플랫폼을 먼저 저장하고, 저장 성공 뒤 씬을 전환한다.
5. 횡단철도에서 지역으로 이동할 때 지역 게이트를 먼저 저장하고, 저장 성공 뒤 씬을 전환한다.

## 항공 허브 진입

1. 기존 전국 월드의 공항 노드는 입국심사 학습과 확장 오버월드 항공편을 함께 제공한다.
2. `releaseEligible=true` 지역만 일반 이용자에게 표시한다. `releaseEligible=false` 미리보기 지역은
   `profiles.role=admin`인 관리자에게만 목적지 버튼을 노출한다.
3. 항공편 선택 시 목적지 지역의 고정 게이트 좌표를 `/api/world/position`에 먼저 저장한다.
4. 저장이 성공한 경우에만 지역 씬을 시작한다. 실패하면 전국 월드에 그대로 남고 재시도 가능한 오류를 표시한다.
5. 디버그 씬 전환은 이 항공 허브 권한·선저장 계약을 대신하지 않는다.

## 현재 제한

- 지역 콘텐츠는 아직 없다. KR/JP 외 지역은 지형·교통만 제공한다.
- 강·철도·지역② 중립 경계는 카메라가 걸친 256타일 저장 청크의 JSON만 불러와 화면 범위 선분만 그린다.
- 1타일 halo의 중복 선분은 전역 segment ID로 제거하며, 레이어별 최근 12개 문서만 메모리에 둔다.
- 지역② 경계는 일반 국제 경계만 절제된 실선, 모든 분쟁·휴전·미확정 분류는 동일 중립 점선으로 표시한다.
  국가 채색·소유권 라벨·원본 지명은 포함하지 않으며 충돌과 이동에는 영향을 주지 않는다.
- 지명 표시는 내부 ID·로케일별 표시명과 언어 앵커 검수 뒤 별도 통합한다.
- 장거리 AOI 계약 전까지 다른 이용자는 지역 씬에서 표시하지 않는다.
- 유럽 지역의 최종 투영·민감 경계 판정 전에는 출시 대상으로 승격하지 않는다.
